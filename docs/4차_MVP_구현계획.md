# 4차 MVP 구현 계획 — 리텐션 루프 (돌아올 이유, 머무를 재미)

> 작성일: 2026-07-11
> 선행 조건: 3차 MVP 완료 ([3차_MVP_구현계획.md](3차_MVP_구현계획.md))
> 목표 기간: 3~4주
> 검증 가설: **"복귀 보상·데일리 루프·살아있는 개성이 있으면 사용자가 매일, 여러 날 연속으로 돌아오는가?"**

---

## 1. 두 제안 목록 비교와 통합 방향

| 관점 | 목록 A (리텐션 훅) | 목록 B (인게이지먼트 깊이) |
|------|------------------|--------------------------|
| 핵심 질문 | **왜 다시 오는가** | **와서 무엇을 하는가** |
| 강점 | 복귀 동기(오프라인 요약, 데일리 리듬, 랜덤성) | 체류 재미(상호작용, 개성, 기록/애착) |
| 4차 채택 | 1·2·4번 전부, 3번은 씨앗만, 5번 이후 | 상호작용(쓰다듬기)·개성(성격/컨디션)·기록(일지) 채택. 미션은 A-2와 통합 |

**통합 원칙**: 4차의 테마는 **리텐션**이다. A가 뼈대(돌아올 이유), B가 살(돌아왔을 때의 재미)이다.
방치형 게임의 하루 루프가 완성되어야 수집/꾸미기(다음 단계)가 의미를 가지므로, 콘텐츠 확장형 항목은 5차로 보낸다.

### 1.1 항목별 결정

| 제안 | 결정 | 이유 |
|------|------|------|
| A-1 오프라인 진행 + 복귀 요약 | ✅ **4차 최우선** | 감쇠 정산은 이미 있음(`applyTimeDecay`). 없는 것은 **긍정적 진행**(부재 중 발견)과 **요약 모달** — 이것이 진짜 공백 |
| A-2 데일리 리듬 + B 데일리 미션/연속 보상 | ✅ 통합 채택 | 출석 스트릭 + 미션 3종으로 통합. 페널티는 기존처럼 없음(죽음/가출 금지 유지). B의 주간 목표는 이후 |
| A-4 랜덤 이벤트 + B 랜덤 행동 | ✅ 채택 (경량) | 결정적 날씨 + 낮잠 + 산책/부재 중 발견. "열 때마다 다른 장면" |
| B 상호작용 (쓰다듬기) | ✅ 채택 | 무료·짧은 상호작용이 데일리 미션의 재료가 됨. 목욕/미니게임은 이후 |
| B 개성 (성격/컨디션) | ✅ 채택 (경량) | 성격 3종(행동 가중치) + 컨디션(배고픔/행복에 따른 속도·표정). 좋아하는 음식은 먹이 1종뿐이라 이후 |
| B 기록과 추억 (성장 일지) | ✅ 채택 | 타임라인은 애착 = 장기 리텐션. 사진 앨범은 이후 |
| A-3 수집/변이 | 🌱 **씨앗만** | 부화 시 껍질 변이(희귀도) 부여 — 이미 있는 `color` 필드 활용. **도감·세대 교체는 5차** (달팽이 1마리 구조에서 도감은 아직 무의미) |
| A-5 공유 카드 | ⏭ 5차 | 일지가 먼저. 캔버스 카드 생성은 다음 |
| B 수집·꾸미기 확장 (장식/식물/스킨) | ⏭ 5차 | 데일리 루프 검증 후 확장해야 해금 조건을 설계할 수 있음 |
| B 미니게임 (숨은 달팽이 찾기 등) | ❌ 이후 보류 | MVP 철학상 핵심 루프 검증 먼저 |

## 2. 범위

### 2.1 포함 (기능 6개)

| # | 기능 | 내용 |
|---|------|------|
| 1 | 복귀 리포트 | 부재 30분+ 복귀 시 "그동안 있었던 일" 모달: 경과 시간, 스탯 변화, 부재 중 발견/사건 1줄 |
| 2 | 부재 중 발견 | 부재 4시간마다 확률로 달팽이가 코인/상추를 주워옴 (긍정적 오프라인 진행) |
| 3 | 데일리 루프 | 출석 스트릭(연속 접속 보너스), 데일리 미션 3종(먹이 2회·산책 1회·쓰다듬기 1회) + 완주 보너스 |
| 4 | 쓰다듬기 | 서식지에서 **달팽이를 직접 터치** → 하트 이펙트 + 행복 상승 (쿨다운 30분). 빈 곳 터치는 기존대로 먹이 |
| 5 | 살아있는 연출 | 결정적 날씨(맑음/비/안개, 비 오면 달팽이 활발), 낮잠(깨우기 가능), 컨디션별 속도·표정, 성격 3종 |
| 6 | 성장 일지 | 부화/레벨업/단계 변화/발견/미션 완주 타임라인 (최근 100건). 부화 시 껍질 변이 기록 포함 |

### 2.2 제외 (4차에서 하지 말 것)

- ❌ 죽음/가출/스탯 페널티 강화 — 가벼운 리듬 유지
- ❌ 도감/세대 교체/달팽이 다중 보유 (5차)
- ❌ 장식·식물·스킨 수집 상점 (5차)
- ❌ 공유 카드/스크린샷 (5차)
- ❌ 주간 목표, 목욕/미니게임
- ❌ 푸시 알림/서버/계정 — 여전히 LocalStorage only
- ❌ 프레임워크/캔버스 도입 — 기존 스택·구조 유지

## 3. 게임 규칙 (수치 — 전부 `GAME.CONFIG`에 추가)

### 3.1 출석 스트릭

- 어제 접속했으면 `streak +1`, 끊기면 1로 리셋 (하루 놓침은 리셋하되 잔소리 없음 — 가벼운 페널티 원칙)
- 접속 보상 = 기본 20 + `min(streak - 1, 10) × 2` 코인 (최대 +20)
- 7일 연속 달성일엔 상추 +3 추가 (7일마다 반복)

### 3.2 데일리 미션 (자정 리셋, 로컬 날짜 키)

| 미션 | 조건 | 보상 |
|------|------|------|
| 밥 챙겨주기 | 먹이 주기 2회 | 코인 +10 |
| 산책 다녀오기 | 산책 1회 | 코인 +10 |
| 쓰다듬어주기 | 쓰다듬기 1회 | 코인 +10 |
| **완주 보너스** | 3개 모두 완료 | 코인 +20, 상추 +1 |

- 보상은 조건 달성 시 **자동 지급 + 토스트** (수령 버튼 없음 — 마찰 최소화)

### 3.3 쓰다듬기

- 달팽이 중심 반경 내 터치 → `happiness +5`, 하트 이펙트. 쿨다운 30분 (쿨다운 중엔 하트만 작게, 효과 없음)
- 낮잠 중 쓰다듬으면 깨어남 (기분 나빠하지 않음)

### 3.4 부재 중 발견

- 부재 4시간마다 1회 판정, 확률 35%, 최대 2건 누적
- 발견물: 코인 5~15 (70%) 또는 상추 1 (30%)
- 복귀 리포트에 "산책하다 코인 12개를 주웠어요!" 형태로 표시 후 지급

### 3.5 날씨 (결정적 — 저장 불필요)

- 날짜 키 해시로 결정: 맑음 60% / 비 25% / 안개 15%
- 비: 달팽이 활발 (idle 시간 0.6배, 이동 속도 1.2배 — 실제 달팽이는 비를 좋아한다), 빗줄기 CSS 오버레이
- 안개: 옅은 안개 오버레이, 행동 변화 없음

### 3.6 성격 (부화 시 랜덤 1개)

| 성격 | 확률 | 행동 가중치 |
|------|------|------------|
| 먹보 | 40% | 먹이 추적 속도 1.3배 |
| 모험가 | 35% | idle 시간 0.7배 (자주 돌아다님) |
| 잠꾸러기 | 25% | 낮잠 확률 2배 |

### 3.7 컨디션 (표정/속도 — 저장값 아님, 스탯에서 파생)

- `hunger ≥ 70`: 이동 속도 0.7배 + 시무룩 표정(더듬이 처짐 SVG 클래스)
- `happiness ≥ 80`: 이동 속도 1.15배 + 밝은 표정
- 낮잠: idle 진입 시 10% 확률(성격 반영), 60~120초, Zzz 이펙트, 시간 감쇠는 그대로

### 3.8 껍질 변이 (수집의 씨앗 — 기존 `color` 필드 활용)

| 변이 | 확률 | 표현 (`--shell-*` 오버라이드) |
|------|------|------------------------------|
| 갈색 (기본) | 55% | 현재 색 |
| 회갈색 | 18% | 차분한 회색조 |
| 적갈색 | 15% | 붉은 기 |
| 올리브 | 10% | 녹색 기 |
| **황금** | 2% | 금색 + 은은한 반짝 연출 |

- 부화 시 결정, 일지에 기록 ("황금빛 껍질을 가지고 태어났어요!")
- 5차 도감/세대 교체의 데이터 기반이 된다

## 5. 도메인 로직 (`game.js` 추가 — 전부 순수 함수)

| 함수 | 역할 |
|------|------|
| `GAME.summarizeAway(snail, player, nowISO, rng)` | 부재 정산 통합: 감쇠 + 발견 판정 → `{ snail, player, report }`. report = 경과시간/스탯변화/발견 목록 |
| `GAME.pet(snail, player, nowISO)` | 쓰다듬기 (쿨다운 검사, happiness) |
| `GAME.applyStreak(player, todayKey)` | 스트릭 갱신 + 접속 보상 계산 (기존 `claimDaily` 대체·확장) |
| `GAME.recordMission(player, kind, todayKey)` | 미션 카운트 + 달성/완주 보상 계산 → `events` |
| `GAME.weatherFor(dateKey)` | 날짜 해시 → 'sunny' \| 'rain' \| 'fog' (결정적) |
| `GAME.rollVariant(rng)` / `GAME.rollPersonality(rng)` | 부화 시 변이/성격 결정 (`hatch`에 통합, `rng` 주입으로 테스트 가능) |
| `GAME.conditionOf(snail)` | 스탯 → 컨디션('hungry' \| 'happy' \| 'normal') + 속도 배수 |

- **랜덤은 전부 `rng` 파라미터 주입** (기본 `Math.random`) — 순수성 유지, 테스트에서 고정 시드 사용
- 기존 함수 시그니처는 유지. `feed`/`walk` 결과에 미션 반영은 호출부(App/HomeModule)가 `recordMission`을 이어 호출

## 6. 화면/모듈 변경

### 6.1 복귀 리포트 모달 (신규 — toast.js 확장 또는 전용 뷰)

```
🐌 다녀오셨어요? (7시간 32분)
─────────────────────────
· 배고픔이 35 올랐어요
· 산책하다 코인 12개를 주웠어요!
· 밖에 비가 와요 — 달달이가 신났어요
              [달달이 보러가기]
```

- 표시 조건: 부재 30분 이상. 부팅 시 `summarizeAway` → 모달 → 닫으면 홈
- 접속 보상/스트릭도 이 모달에 통합 표시 (토스트 난사 방지)

### 6.2 홈 (서식지)

- 날씨 오버레이 레이어 (CSS 애니메이션: 빗줄기/안개)
- 달팽이 터치 판정: 터치 지점이 달팽이 반경 내면 쓰다듬기, 밖이면 먹이 드롭 (`HabitatModule` pointerdown 분기)
- 하트/Zzz/반짝(황금 변이) 이펙트는 기존 `float-text` 패턴 재사용
- 좌하단 미션 칩: `오늘의 돌봄 2/3` — 탭하면 미션 시트(오버레이) 표시

### 6.3 달팽이 탭 (구 스탯 탭 확장)

- 상단: 기존 스탯 + **성격/변이 표시** ("모험가 · 올리브 껍질")
- 하단: **성장 일지 타임라인** (최근 순, 날짜 구분선)

### 6.4 모듈 구성

## 8. 완료 조건

1. 30분+ 부재 후 복귀하면 경과 요약 모달이 뜨고, 발견물이 정확히 지급된다
2. 연속 접속 시 보상이 커지고, 끊겨도 잔소리 없이 조용히 1일차부터 다시 시작한다
3. 데일리 미션 3종이 자정(로컬)에 리셋되고, 달성 즉시 자동 지급된다
4. 달팽이를 터치하면 쓰다듬기(하트), 빈 곳을 터치하면 먹이 드롭으로 정확히 구분된다
5. 비 오는 날 달팽이가 눈에 띄게 활발하고, 낮잠을 자다 쓰다듬으면 깬다
6. 성격에 따라 행동 패턴이 다르고, 배고프면 느리고 시무룩해진다 (죽지는 않는다)
7. 부화 시 변이가 결정되어 껍질 색이 다르고, 황금 변이는 반짝인다
8. 달팽이 탭에서 성격/변이와 성장 일지 타임라인을 볼 수 있다
9. v2 저장 데이터가 마이그레이션되어 그대로 이어진다 (성격은 기존 달팽이에도 1회 부여)
10. 모든 랜덤이 `rng` 주입식이라 테스트에서 재현 가능하고, 통합 테스트 전체 통과 + 콘솔 에러 0개

## 9. 5차 후보 (이번에 의도적으로 미룬 것)

1. **도감 + 세대 교체**: 성체를 "여행 보내기" → 앨범에 기록 → 새 알(변이 확률 상승). 변이 수집의 완성
2. 꾸미기 확장: 장식/식물/돌 — 미션·성장으로 해금
3. 공유 카드 (캔버스로 달팽이 카드 이미지 저장)
4. 사진 앨범 / 첫 행동 기록 강화
5. 주간 목표, 목욕·미니게임

> 5차의 전제는 "4차의 데일리 루프가 실제로 재접속을 만드는가"의 검증이다. 수집은 돌아오는 사용자에게만 의미가 있다.
