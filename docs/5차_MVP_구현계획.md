# 5차 MVP 구현 계획 — 수집의 완성 & 게임필(Game Feel)

> 작성일: 2026-07-11
> 선행 조건: 4차 MVP 완료 ([4차_MVP_구현계획.md](4차_MVP_구현계획.md))
> 목표 기간: 3~4주
> 검증 가설: **"세대를 거듭할 이유(도감)와 손맛(피드백)이 있으면 '오래 하는 게임'이 되는가?"**

---

## 1. 개요

### 1.1 5차의 두 축

| 축 | 내용 | 출처 |
|----|------|------|
| **A. 수집의 완성** | 세대 교체(여행 보내기) + 앨범/도감 — 4차에 심은 변이 씨앗을 장기 목표로 완성 | 4차 계획 §9의 1순위 |
| **B. 게임필(Game Feel)** | 효과음, 코인 플라이, 카운트업, 전환 애니메이션 — "게임답게 반응하는 느낌" | 사용자 질문("게임 느낌이 들려면 디자인 수정?")에 대한 답 |

### 1.2 디자인 방향에 대한 결정 (질문에 대한 답)

**아트(그림)는 다시 갈아엎지 않는다.** 3차 실사풍 SVG + 자연 톤은 이 게임의 정체성으로 유지한다.
"게임 느낌"이 부족하다면 그 원인은 그림이 아니라 **반응성**이다:

- 지금: 행동하면 토스트 텍스트가 뜬다 (정보 전달은 되지만 손맛이 없다)
- 목표: 행동하면 **소리가 나고, 코인이 지갑으로 날아가고, 숫자가 굴러 올라가고, 화면이 미세하게 반응**한다

즉 5차의 디자인 작업은 리디자인이 아니라 **주스(juice) 계층 추가**다. 아트 스타일 변경은 하지 않는다.

## 2. 범위

### 2.1 포함 (기능 5개)

| # | 기능 | 내용 |
|---|------|------|
| 1 | 여행 보내기 (세대 교체) | 성체 Lv12+ 달팽이를 여행 보냄(확인 모달 필수) → 앨범 기록 → 새 알 + 세대 +1 |
| 2 | 앨범 & 도감 | 앨범: 역대 달팽이 카드(이름/변이/성격/함께한 날/최종 레벨). 도감: 변이 5종 수집 그리드(미발견은 실루엣) |
| 3 | 세대 보너스 | 세대가 오를수록 희귀 변이 확률 상승 — "다음엔 뭐가 나올까"의 실체 |
| 4 | 꾸미기 확장 (경량) | 장식 4종(돌/버섯/들꽃/이끼바위)을 서식지 고정 슬롯에 배치. 코인 구매 2종 + 해금 2종(미션 누적/세대) |
| 5 | 게임필 폴리시 | 합성 효과음(온오프 가능), 코인 플라이, 숫자 카운트업, 탭 전환 애니메이션, 부화/단계 변화 컨페티 |

### 2.2 제외 (5차에서 하지 말 것)

- ❌ 아트 스타일 리디자인 (실사풍 유지)
- ❌ 외부 사운드 파일/에셋 — 효과음은 **Web Audio API 오실레이터 합성**만 (스택 제약 유지)
- ❌ 공유 카드(캔버스 PNG) — 6차 후보 (앨범이 먼저 자리 잡아야 공유할 것이 생긴다)
- ❌ 달팽이 동시 다중 보유 — 세대는 항상 1마리씩 교대
- ❌ 주간 목표, 미니게임, 서버/계정/푸시

### 2.3 기술 확인

- Web Audio API·`navigator.vibrate`는 브라우저 내장 — 새 의존성 아님. iOS 사파리는 vibrate 미지원이므로 있으면 쓰고 없으면 조용히 건너뛴다
- 효과음은 첫 사용자 제스처 후에만 AudioContext를 생성한다 (자동재생 정책)

## 3. 게임 규칙 (수치 — `GAME.CONFIG` 추가)

### 3.1 여행 보내기 (세대 교체)

| 항목 | 값 |
|------|-----|
| 조건 | `stage === 'adult'` && `level ≥ 12` |
| 절차 | 달팽이 탭의 [여행 보내기] → 확인 모달("영영 이별이 아니라 앨범에 남아요") → 작별 연출 |
| 기념 보상 | 코인 +100 |
| 결과 | 앨범에 기록 → 새 알 지급(`generation +1`) → 이름 짓기 온보딩 재진입 |

- 스탯/코인/상추/스트릭/미션은 유지된다 (잃는 건 없고, 달팽이만 앨범으로 이동)
- 여행 보낸 날 일지에 기록: "달달이(1세대)가 넓은 세상으로 여행을 떠났어요."

### 3.2 세대 변이 보너스 (희귀 확률 상승)

| 변이 | 1세대 | 세대당 변화 | 상한(6세대+) |
|------|-------|------------|--------------|
| 갈색 | 55% | -6%p | 25% |
| 회갈색 | 18% | +1%p | 23% |
| 적갈색 | 15% | +1.5%p | 22.5% |
| 올리브 | 10% | +2%p | 20% |
| **황금** | 2% | +1.5%p | 9.5% |

- 구현: `GAME.variantTableFor(generation)`이 세대 보정된 확률 테이블 반환 → `rollVariant(rng, generation)`
- 도감 완성(5종 수집) 시 칭호 표시("달팽이 박사") — 추가 보상은 없음 (완성 자체가 보상)

### 3.3 장식 (꾸미기 확장)

| 장식 | 획득 | 가격/조건 |
|------|------|-----------|
| 조약돌 | 상점 구매 | 코인 50 |
| 버섯 | 상점 구매 | 코인 80 |
| 들꽃 | 해금 | 미션 완주 누적 7회 |
| 이끼 바위 | 해금 | 2세대 도달 |

- 서식지 고정 슬롯 3곳(좌하/중하/우하) 중 선택 배치, 슬롯당 1개, 언제든 교체/제거
- 장식은 순수 시각 요소 — 게임 수치에 영향 없음 (밸런스 오염 방지)
- 표현은 인라인 SVG (달팽이와 동일 방식)

### 3.4 효과음 (합성 — 파일 없음)

| 소리 | 트리거 | 합성 스펙(대략) |
|------|--------|----------------|
| 뽁 | 버튼/탭 전환 | 短 sine 220→180Hz, 60ms |
| 아삭 | 먹기 완료 | noise burst + lowpass, 90ms |
| 띠링 | 코인 획득 | sine 880→1320Hz 2음, 160ms |
| 팡파레 | 레벨업/부화/여행 | 3음 아르페지오, 400ms |
| 하트 | 쓰다듬기 | soft triangle 660Hz, 80ms |

- 설정 탭에 소리 온/오프 토글 (`sn_player.sound_on`, 기본 켬). 진동(안드로이드)도 동일 토글에 묶음

### 3.5 게임필 연출

- **코인 플라이**: 코인 획득 시 발생 지점 → 지갑 칩으로 코인 아이콘 2~3개가 곡선 비행 후 지갑이 통통 튀며 숫자 카운트업
- **카운트업**: 지갑/스탯 숫자가 즉시 바뀌지 않고 0.4초간 굴러감
- **탭 전환**: 화면이 살짝 페이드+슬라이드 (150ms, `prefers-reduced-motion` 존중)
- **컨페티**: 부화/단계 변화/여행 보내기 때 CSS 파티클 12~16개 (라이브러리 없이)

## 4. 데이터 모델 변경 (`schema_version: 4`)

| 키 | 변경 | 내용 |
|------|------|------|
| `sn_player` | 필드 추가 | `generation`(현재 세대, 기본 1), `sound_on`(기본 true), `mission_completions`(완주 누적 횟수), `decorations: { owned: [], slots: [null, null, null] }` |

| `sn_journal` | 유지 | 여행/해금 등 이벤트 추가 기록 |

- 기존 v3 레코드는 `_getOrInit` 병합으로 자동 치유 (generation 1, 도감은 현재 달팽이 변이부터 시작)
- 도감 발견 목록은 별도 키 없이 **앨범 + 현재 달팽이에서 파생** (중복 저장 금지)

## 5. 도메인 로직 (`game.js` 추가 — 순수 함수)

| 함수 | 역할 |
|------|------|
| `GAME.canGraduate(snail)` | 여행 보내기 가능 여부 (성체 && Lv12+) |
| `GAME.graduate(snail, player, nowISO)` | 앨범 레코드 생성 + 기념 코인 + 새 알 + `generation +1` → `{ snail(새 알), player, record, events }` |
| `GAME.variantTableFor(generation)` | 세대 보정 변이 확률 테이블 (§3.2) |
| `rollVariant(rng, generation)` | 기존 함수 확장 (하위 호환: generation 생략 시 1세대) |
| `GAME.discoveredVariants(album, snail)` | 도감 파생: 발견한 변이 id 목록 |
| `GAME.decorationDefs()` / `GAME.canUnlockDecoration(def, player)` | 장식 정의와 해금 판정 |

- `hatch`는 `generation`을 받아 변이 확률에 반영하도록 시그니처 확장 (기존 호출 하위 호환)
- 사운드는 도메인이 아니라 연출 — `game.js`에 넣지 않는다

## 6. 화면/모듈 변경

### 6.1 달팽이 탭 (앨범/도감 추가)

```
[달팽이 탭]
├─ 현재 달팽이 (기존: 스탯/특성/다음 단계)
│   └─ [🧳 여행 보내기] 버튼 (조건 충족 시 활성, 미충족 시 조건 안내)
├─ 📖 도감 — 변이 5종 그리드 (발견: 색상+이름 / 미발견: 실루엣+???)
├─ 🏆 앨범 — 역대 달팽이 카드 목록 (세대순)
└─ 성장 일지 (기존)
```

### 6.2 꾸미기 탭 (장식 추가)

- 기존 배경 선택 아래 "장식" 섹션: 보유/미보유(가격 또는 해금 조건 표시) + 슬롯 3개 배치 UI
- 서식지에 배치된 장식은 `#deco-layer`(신규, z-index: 식물 실루엣과 달팽이 사이)에 SVG로 렌더

### 6.3 신규/변경 모듈

| 모듈 | 변경 |
|------|------|
| `js/sound.js` (신규) | `Sound` — Web Audio 합성 효과음 + 토글. 어디서든 `Sound.play('coin')` |
| `js/fx.js` (신규) | `FX` — 코인 플라이/카운트업/컨페티 (DOM 연출 전담) |
| `stats.js` | 도감/앨범/여행 보내기 섹션 렌더 |
| `deco.js` | 장식 구매/해금/슬롯 배치 |
| `habitat.js` | `#deco-layer` 렌더 (배치 슬롯 → SVG) |
| `db.js` | schema v4, `DB.Album` 신설 |
| `settings.js` | 소리/진동 토글 |
| `app.js` | 탭 전환 애니메이션 클래스, 효과음 트리거 연결 |

- 효과음/연출 호출은 이벤트 처리 지점(handleResult/_showEvents 등)에 연결 — 도메인 로직과 분리 유지

## 8. 완료 조건

1. 성체 Lv12 달팽이를 여행 보내면 확인 모달 → 작별 연출 → 앨범 기록 → 새 알 이름 짓기로 자연스럽게 이어진다
2. 2세대 알부터 희귀 변이 확률이 §3.2 표대로 상승한다 (시드 고정 테스트로 검증)
3. 도감에서 발견한 변이는 색상으로, 미발견은 실루엣+???로 표시된다
4. 앨범에 역대 달팽이의 이름/변이/성격/함께한 기간/최종 레벨이 남는다
5. 장식 4종을 구매/해금해 서식지 슬롯에 배치·교체할 수 있고 재실행 후 유지된다
6. 코인 획득 시 코인 플라이 + 카운트업, 부화/단계 변화/여행 때 컨페티가 나온다
7. 효과음이 파일 없이 합성으로 재생되고, 설정에서 끄면 완전 무음이다
8. `prefers-reduced-motion` 설정 시 플라이/컨페티/전환 애니메이션이 비활성화된다
9. v3 저장 데이터가 그대로 이어진다 (generation 1로 시작, 현재 달팽이가 도감 첫 발견)
10. 통합 테스트(세대 교체 → 새 알 → 도감 갱신 흐름 포함) 전체 통과 + 콘솔 에러 0개

## 9. 6차 후보 (이번에 의도적으로 미룬 것)

1. 공유 카드 — 앨범 카드를 캔버스로 PNG 저장 (앨범이 생겼으니 다음엔 자연스럽다)
2. 무늬 변이 2차원 수집 (색 × 무늬 조합)
3. 목욕/미니게임 등 상호작용 확장
4. 주간 목표 / 시즌성 이벤트
5. PWA(홈 화면 설치 + 오프라인 캐시) — "앱처럼" 배포의 다음 단계
