# 13차 MVP 구현 계획 — 진짜 앱으로서의 배포 & 수집·공유 심화

> 작성일: 2026-07-13
> 선행 조건: 12차(탭 개편·설정 분리·도감 등급·미니게임 전환) 완료 — v1.4.0 배포됨
> 목표 기간: 4~5주
> 검증 가설: **"설치해서 쓰는 '진짜 앱'이 되고, 보여주고 모을 게 늘면 유저가 남고 남에게 권하는가?"**
> 성격: 11~12차가 콘텐츠·UX였다면, 13차는 **"제품으로서의 완성"**(설치·계정·법적 고지·스토어) + 수집/공유 심화.

---

## 0. 배경

- 1~7차: 코어 케어 루프 → 멀티/탐험/양육자
- 8~10차: 서버 전환 + 운영 기반(원격 설정·이벤트·공지·어드민·CD)
- 11차: 살아있는 생태계 + 슬롯 8마리
- 12차: 탭 개편(유저/도감/홈/미니게임/상점) + 설정 분리 + 미니게임 허브(탐험·경주) + 도감 등급
- 현재 변이: 기본 10 + 레어(연못). 스페셜/에픽 슬롯은 비어 있음(RARITIES에 정의만)

**13차는 두 갈래다.** (A) **제품화** — PWA 설치·계정·법적 고지로 "진짜 앱"이 된다. (B) **콘텐츠 심화** — 공유·히든 변이·미니게임 퀴즈로 리텐션을 올린다. A가 우선순위다(배포 목표).

---

## A. 제품화 — 진짜 앱으로서의 배포

### A.1 PWA 설치 완성도
- **매니페스트 점검**: `manifest.json` 아이콘(192/512/maskable), `theme_color`/`background_color`, `display: standalone`, `scope`/`start_url`.
- **아이콘·스플래시**: `assets/icon-*` 재정비. iOS `apple-touch-icon`, 안드로이드 maskable 여백.
- **설치 유도(A2HS)**: `beforeinstallprompt` 캐치 → 설정/홈에 "앱 설치" 버튼. iOS는 "공유 → 홈 화면에 추가" 안내 모달.
- **오프라인 점검**: sw 캐시 범위·`/v1/*` 미캐시 재확인, 오프라인 첫 진입 UX.

### A.2 계정 — 게스트 → 소셜 연결 UI
- 서버는 게스트/Google 연결 API가 이미 있음(8차). **프론트 UI만** 추가: 설정 → 계정 → "Google로 로그인/연결", 연결 상태 표시, 기기 이전 안내.
- 게스트 데이터 유지한 채 소셜 연결(서버 원칙 준수). 충돌(이미 연결된 소셜) 시 안내.
- 백업 코드는 보조 수단으로 유지.

### A.3 법적 고지 & 스토어 준비
- **개인정보 처리방침 / 이용약관**: 정적 페이지(`docs/legal/` 또는 설정 내 링크). 수집 항목(닉네임·기기 토큰)·서버 위치 명시.
- **정보 화면**(설정 §추천): 버전·크레딧(아트/사운드)·문의처·오픈소스 고지.
- **스토어 배포 경로 결정**(택1, 조사·문서화): PWA 설치형 / TWA로 Google Play / Capacitor 래핑.
- 스토어 자산: 스크린샷·설명·아이콘 규격.

### A.4 온보딩 개선
- 첫 실행(알 → 이름짓기)에 **1~2장 안내**(무엇을 하는 게임인지, 하루 2번 돌봐요) 추가.
- 서버/오프라인 모드 표시 명확화.

### A.5 관측·안정성(운영 준비)
- 프론트 오류 수집(선택: Sentry 프론트) — 9차 백엔드 Sentry와 연결.
- 배포 스모크에 핵심 유저 플로우(게스트 생성→부화→급식) E2E 1개 추가.

## B. 콘텐츠 심화 — 공유·수집·미니게임

### B.1 공유 카드
- 서식지(디오라마)를 `<canvas>`로 캡처(무의존 — PNG 스프라이트 직접 draw). 배경+개체(색/단계/포즈)+이름/세대/도감 진행률+워터마크.
- `canvas.toBlob` → 저장 / `navigator.share`(지원 시). 동일 출처라 canvas 오염 없음.
- 접근: 유저 탭 또는 홈 롱프레스 → "디오라마 카드".

### B.2 히든 변이 — 스페셜/에픽 (아트 완성 후)
- `docs/art/characters/미완성/`(악마·천사·재벌·충무공) **세트 완성(아기/주니어/성체 3종) 후** 추가.
- 등급·등장 조건은 **원격 게임 설정(10차)**으로: 예) 악마=밤 부화 저확률, 천사=도감 완성률, 재벌=코인 누적, 충무공=기간 이벤트. 확률 합=1 검증 재사용.
- 부화 연출: epic 강조(컨페티·사운드·도감 배지). `GAME.spritePath` 폴백으로 무효 변이 404 차단.

### B.3 미니게임 — 달팽이 퀴즈
- 문항 데이터(달팽이/게임 상식) + 정답 시 소액 코인, 하루 제한(경주와 동일 패턴: `minigame_*` 상태·서버 판정 `POST /v1/minigame/quiz`).
- 경주(12차 완료)와 같은 서버 권위 구조 재사용.

### B.4 도감 심화 & 우정 지표
- 등급 완성 보상(코인/장식/타이틀) — 우편함 재사용, 수령 멱등.
- "달팽이 박사" 타이틀 등급별 확장(레어 컬렉터/에픽 헌터).
- 11차 사회 행동 데이터로 "단짝" 표시(연출, 저장 소폭).

## D. 선행 필요 (사용자)

- **히든 변이 아트 완성**(악마/천사/재벌/충무공 3단계 세트) — 없으면 7단계 보류.
- **스토어 배포 정책 결정**: PWA 설치형 vs Google Play(TWA) — 계정/개인정보 요구가 달라짐.
- **Google OAuth 클라이언트**(계정 연결 프론트) — 서버 `GOOGLE_CLIENT_ID`는 준비됨.

---

## E. 완료 조건 (안)

1. 홈 화면에 설치할 수 있고(A2HS), 설치본이 오프라인에서 정상 실행된다
2. 설정에서 Google 계정을 연결하고 상태가 표시된다(게스트 데이터 유지)
3. 개인정보/약관/정보 화면이 있고 스토어 자산이 준비된다
4. 첫 실행 온보딩이 게임을 설명한다
5. 서식지를 이미지 카드로 저장/공유할 수 있다
6. 퀴즈 미니게임이 서버 판정·하루 제한으로 동작한다
7. (아트 완성 시) 스페셜/에픽 변이가 조건부 등장한다
8. 기존 기능 무회귀, E2E 스모크 통과, v2.0.0 정식 배포
