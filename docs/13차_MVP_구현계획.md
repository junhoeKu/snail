# 13차 MVP 구현 계획 — 공유·수집 심화 & 히든 변이 (계획만)

> 작성일: 2026-07-13
> 선행 조건: 12차(탭 개편·설정 분리·도감 등급·미니게임 전환) 완료
> ⚠️ **12차에서 13차로 이월**: 12차가 탭/UX 개편으로 바뀌면서 공유·히든변이는 13차로 밀림
> 상태: **계획 수립 단계** (구현 미착수). 11차 §10 차기 후보 + 12차 미니게임(경주·퀴즈)을 정리한다.
> 목표 기간: 3~4주
> 검증 가설: **"내가 키운 디오라마를 남에게 보여주고 싶고, 희귀한 걸 모으고 싶어 다시 들어오는가?"**

---

## 0. 배경 — 지금까지의 토대

- 1~7차: 코어 케어 루프 → 멀티/탐험/양육자
- 8~10차: 서버 전환 + 운영 기반(원격 설정·이벤트·공지·어드민·CD)
- 11차: 살아있는 생태계(행동 선택기·사회 행동·부재 생활) + 슬롯 8마리
- 색 팔레트: 기본 10종(갈색·회갈색·붉은색·노란색·블루그레이·라벤더그레이·허브·검정·라임·소라) + 레어 1종(연못). 등급 확장(스페셜/에픽)은 미사용 슬롯으로 남겨둠

**12차는 "보여주기(공유)"와 "모으기(수집 심화)"** 두 축으로 리텐션을 끌어올린다. 원격 설정·이벤트 인프라가 이미 있으므로 확률·등장은 재배포 없이 조정 가능하다.

## 1. 범위 (11차 §10 차기 후보 반영)

| # | 기능 | 요약 |
|---|---|---|
| 1 | **공유 카드** | 현재 서식지(디오라마)를 이미지 카드로 캡처해 저장/공유 |
| 2 | **히든 변이(스페셜/에픽)** | 미완성 아트(악마/천사/재벌/충무공) 완성 시 조건부 등장 |

## 2. 공유 카드 (§1)

### 2.1 목표
"내 디오라마 한 장"을 남에게 보여주는 자연스러운 바이럴 후크.

### 2.2 설계 (무빌드 제약 안에서)
- **캡처**: 서식지 DOM을 캔버스로 렌더. 외부 라이브러리 없이 하려면 `<canvas>`에 배경(assets/backgrounds) + 달팽이 스프라이트 + 장식을 좌표대로 다시 그린다(html2canvas 같은 의존성 회피 — 스프라이트가 PNG라 직접 draw 가능).
- **카드 구성**: 배경 + 현재 개체들(색/단계/포즈) + 이름/세대/도감 진행률 + 워터마크(앱 주소).
- **저장/공유**: `canvas.toBlob` → 다운로드 / Web Share API(`navigator.share`, 지원 시 이미지 공유).
- **접근**: 설정 탭 또는 홈 롱프레스 → "디오라마 카드 만들기".

### 2.3 주의
- 스프라이트 CORS: 동일 출처(GitHub Pages)라 canvas 오염 없음. 서버 모드여도 이미지는 정적 자산.
- 성능: 캡처는 1회성이라 rAF 루프와 무관. 캡처 시 애니메이션 일시정지 후 그린다.

## 3. 히든 변이 — 스페셜/에픽 (§2)

### 3.1 미완성 아트 완성 전제
`docs/art/characters/미완성/`에 악마·천사·재벌·충무공 일부만 있음(세트 미완). **완성 후** 추가한다. 세트 완결(아기/주니어/성체 3종) 전엔 등장 금지.

### 3.2 등급·등장 조건 (안)
| 변이 | 등급 | 등장 조건(예시 — 원격 설정으로 조정) |
|------|------|-----------------|
| 악마 | epic | 밤에 부화 + 낮은 확률 / 특정 이벤트 |
| 천사 | epic | 도감 일정 완성률 + 낮은 확률 |
| 재벌 | special | 코인 누적/특정 세대 |
| 충무공 | special | 기간 한정 이벤트 전용 |

- `RARITIES`에 이미 정의된 등급 체계 재사용(common/rare + special/epic 추가).
- 확률·조건은 **원격 게임 설정**(10차)으로 넣어 재배포 없이 튜닝. 변이 확률 합=1 검증은 기존 검증기 재사용.
- 부화 연출: epic은 강한 컨페티+전용 사운드, 도감에 등급 배지 강조.

### 3.3 데이터
- 변이 추가는 `VARIANTS`/`VARIANT_GEN_DELTA`(클라·서버 미러) + 스프라이트 27→+N. 확률 재분배는 이번 색 추가와 동일 절차.
- `GAME.spritePath` 폴백이 이미 있어 무효 변이 404는 구조적으로 차단됨.

## 4. 도감 심화 (§3)

- 등급별 섹션 분리(기본/레어/스페셜/에픽), 미발견은 실루엣.
- 완성 보상: 등급 완성 시 코인/장식/타이틀. 서버 권위(수령 멱등 — 우편함 재사용).
- "달팽이 박사" 타이틀을 등급별로 확장(예: 레어 컬렉터, 에픽 헌터).

---

## 5. 개체 간 우정 지표 (§4)

- 11차 사회 행동(인사/같이 쉬기) 발생 시 개체 쌍 카운트를 누적(연출 계층, 저장 소폭).
- 팝업/유저 탭에 "단짝: 몽이" 표시. 게임 수치 무영향(연출).
- 저장: `sn_snails`에 개체별 친밀 맵 소폭 추가 or 별도 키.

## 8. 완료 조건 (안)

1. 서식지를 이미지 카드로 저장/공유할 수 있다.
2. 도감이 등급별로 나뉘고, 등급 완성 보상이 우편함으로 지급된다(멱등).
3. (아트 완성 시) 스페셜/에픽 변이가 조건부로 등장하고 도감에 등급이 강조된다.
4. 자주 어울린 개체 쌍이 "단짝"으로 표시된다.
5. 기존 기능(성장/경제/생태계/서버 판정) 무회귀 — 신규는 전부 연출/수집 계층.
