## Phase 1 — 핫픽스·UX 정리 (P0 · 공수 ~1일 · `feat/v13-hotfix`)

**왜 이 단계가 먼저인가**: 셋 다 사용자가 매 세션 겪는 체감 마찰이고, 전부 CSS 수정만으로 끝나 의존성이 0이다. 하루 안에 릴리스해 신뢰를 회복한다.

### 1.1 토스트 클릭 차단 수정 (#1)
- `components.css` `#toast-container`에 `pointer-events: none;` 추가. 토스트는 인터랙션이 없으므로 부작용 없음.
- 회귀 확인: `.modal-overlay`(z-index 200)는 의도적 차단이므로 유지.

### 1.2 경주 예측 버튼 1줄 고정 (#2)
- `.race-guess { flex-wrap: nowrap; }` + `.race-pick { flex: 1 1 0; min-width: 0; padding: 10px 4px; }`.
- race.js는 변경 불필요(버튼은 JS 생성이지만 class만 사용). 320px 폭에서 5버튼 확인.

### 1.3 달팽이 크기 축소 (#7)
- `.snail-img` 44/64/84 → **36/52/68px**(약 -20%). 성체 기준 서식지 폭 대비 ~22% → ~17%.
- `_edge()`·머리 위 배지·이모트 오프셋은 `offsetWidth` 기반이라 자동 추종. `PET_RADIUS_MIN: 34`는 유지(터치 판정 최소 보장).
- 경주 스프라이트(`.race-snail` 40px)는 별도 — 레인 높이와 균형 유지, 변경 없음.

**릴리스**: `sw.js CACHE_VERSION` 범프 필수(css 변경). 릴리스는 `python3 scripts/release.py` 경유.

## Phase 2 — 먹이 지속: 서버 저장 (P1 · 공수 ~1주 · `feat/v13-food-persist`) (#3)

**왜 Phase 2인가**: "먹이 주기"는 코어 루프의 첫 행동인데 결과가 증발하면 케어 게임의 신뢰가 깨진다. 백엔드가 배포되어 있어 즉시 가능하고, Phase 1과 코드 영역이 겹치지 않지만 서버 스키마 작업이 있어 별도 브랜치로 분리한다.

### 설계 원칙
- **드롭 = 상태 기록, 차감·보상 = 기존 feed 경로 유지.** 게임 수치 판정은 바뀌지 않으므로 `rules.py` 무변경. 재고 차감은 지금처럼 먹기 완료 시 `service.add_item`(원장) 경유.

### 서버 (backend)
- `User.dropped_foods` JSON 컬럼 추가 — Alembic **expand 마이그레이션**(방어적·idempotent, CI 왕복 검사 통과).
- 항목: `{ id, food_id, rx, ry, dropped_at }` — 좌표는 서식지 `pos` 관례대로 **비율(rx/ry)** 저장.
- API (`modules/actions.py` 인접):
  - `POST /v1/habitat/foods` — 드롭 등록. `request_id` 멱등, 재고 **사전 검증만**(차감 없음), 상한 10(`FOOD_MAX`와 동일 값 — rules 상수로 승격) 검증.
  - 먹기 완료: 기존 `POST /snails/{id}/feed` body에 `drop_id` 추가 → 성공 시 해당 드롭 제거.
  - `GET /v1/state` 응답에 `dropped_foods` 포함(부팅 복원용).
- TTL: 24시간 지난 드롭은 lazy 정리(시간 판정 관례 준수 — 배치 없음).

### 클라이언트
- `habitat.js dropFood()` 성공 시 `Api.dropFood()` 등록 — 실패(network)는 기존 오프라인 큐(`Api.queueFeed` 패턴) 재사용.
- 부팅/`sync()` 시 서버 `dropped_foods` → `_foods` 복원 → 기존 `_assignFoods()`가 자동으로 이어 먹기 시작(추가 로직 불필요).
- **로컬 모드 대칭**: `DB.Player`에 `dropped_foods` 저장(스키마 버전 업 + 마이그레이션) — 서버/로컬 동작 동일.

### 엣지 케이스
- 드롭 후 다른 기기 접속: state 복원으로 자연 해결. 같은 드롭 이중 섭취는 `drop_id` + 행동 멱등키가 방지.
- 응답 필드 추가만 있고 제거 없음 → 구버전 클라 호환(Expand 단계만으로 완결).

## Phase 3 — 상호작용 확장: 모습 전환 + 드래그 (P1~P2 · 공수 ~1주 · `feat/v13-interaction`)

**왜 묶는가**: 둘 다 home.js 팝업/habitat.js 포인터 계층을 만지는 작업이라 충돌을 피하려면 한 브랜치가 낫다. Phase 1·2보다 설계 여지가 커서 뒤에 둔다.

### 3.1 모습 바꾸기 — 팝업 안 버튼 (#6, 사용자 확정)
- 달팽이 팝업(`home.js openSnailPopup`)에 **[모습 바꾸기]** 버튼 → 현재 단계 **이하**(도달한 단계)만 선택 가능한 선택지 표시. 성체가 아기 모습으로 회귀 가능.
- 저장: `snail.skin_stage` (null = 실제 stage 그대로).
  - 서버: 컬럼 expand + `PATCH /v1/snails/{id}/skin` (검증: 도달 단계 이하만).
  - 로컬 모드: `DB.Snails.saveOne` 경유.
- 표시: `_applyLook()`과 `GAME.spritePath(color, skin_stage || stage)` — **연출 전용**. 레벨업/도감/스탯 판정은 실제 `stage` 유지(rules.py 무영향).
- 단계 변화(진화) 시 `skin_stage` 초기화 — 새 모습을 먼저 보여준다.

### 3.2 드래그 이동 — 평가 결론: **채택 권장 (저위험·중공수)** (#4)
- **충돌 분석**: 현재 `pointerdown` 1개로 [달팽이 근처=팝업 / 빈 곳=먹이드롭] 분기. 드래그를 넣으려면 이동 임계값(~8px) 전=탭, 후=드래그로 판정 분리 필요 — `pointermove`/`pointerup` + `setPointerCapture` 표준 패턴.
- **상태머신 간섭**: rAF 루프(`_updateEnt`)가 드래그 중 위치를 되돌림 → `DRAGGING` 상태를 추가해 해당 개체만 루프에서 제외, 놓으면 `_savePosition` + `IDLE`. 저장은 기존 `pos`(rx/ry) 재사용이라 **서버 작업 불필요**.
- **정체성 검토**: "스스로 살아가는 디오라마"와 상충할 수 있으나, 집어들면 놀람 이모트(💦)+껍질 숨기, 놓으면 잠깐 두리번 후 일상 복귀로 연출하면 오히려 애착 상호작용이 된다.
- **결론**: 클라 전용 변경, 기존 저장 구조 재사용, 되돌리기 쉬움 → 채택. 단 Phase 3에서 프로토타입 → 자체 QA 후 릴리스 판단.

## Phase 4 — 살아있는 맵: 리디자인 + 동적 날씨·이벤트 (P2 · 공수 2~3주 · `feat/v13-living-map`) (#5)

**왜 마지막 대형 단계인가**: 배경 일러스트 등 **아트 자산 의존**(release.py 파이프라인으로 반입)이고, 클라 연출 + 서버(원격 설정) 양쪽을 만져 공수가 가장 크다. 기능적으로는 다른 Phase에 의존하지 않아 아트 준비와 병행 가능하다.

### 4.1 맵 리디자인
- 배경 일러스트 교체 + **전경 레이어**(풀숲/돌 실루엣, z-index로 달팽이 앞) 추가 → 깊이감. Phase 1의 크기 축소와 합쳐 "넓은 정원" 공간감.
- 장식 슬롯(`DECO_SLOTS`) 3 → 5 확대 검토(앵커 행동 다양화).
- 아트 반입은 `scripts/release.py` + manifest 매핑 관례 준수(macOS NFD 주의).

### 4.2 동적 날씨 (하루 안에서 변화)
- 현재 일 단위 해시 → **(dateKey + 시간 슬롯) 해시**로 확장: 하루 3~4구간, 결정적이라 저장 불필요·서버/클라 동일 판정 관례 유지. `game.js weatherFor`와 `rules.py`를 **대칭으로** 수정하고 가중치는 원격 게임 설정(10차)으로 조정 가능하게.
- 비 연출 강화: 빗방울 파티클(FX 확장) + 바닥 물웅덩이, **비 갠 직후 무지개**(레어 연출 — 스크린샷 유도), 밤(`data-daytime`) 반딧불이.

### 4.3 동적 이벤트 (소규모)
- 방문객: 나비/무당벌레가 가끔 지나감 — 엔티티 이동 로직 재사용, 순수 연출.
- 클로버 스폰: 드물게 바닥에 등장, 탭하면 소액 코인 — **보상이 있으므로 서버 판정**(minigame 패턴·일일 상한 재사용).

## Phase 5 — 힐링·몰입 백로그 (P3 · 상시 · 릴리스 틈새 1~2개씩) (#8)

**왜 백로그인가**: 개별 공수는 작지만(각 0.5~2일) 버그·코어 루프보다 앞설 이유가 없고, 아트/사운드 자산 준비에 따라 순서가 유동적이다. 전부 연출 전용 — `GAME.CONFIG`/rules.py 무영향.

| 제안 | 내용 | 공수 |
|------|------|------|
| 표정 시스템 | 컨디션별 눈/입 오버레이(`cond-*` 클래스 확장) — 배부름😊/배고픔😢/졸림😪 | 소 (아트 필요) |
| 점액 트레일 | 이동 경로에 옅은 자국이 잠시 남았다 페이드 | 소 |
| 유휴 모션 추가 | 하품·스트레칭·두리번 — `BEHAVIOR` 가중치에 항목 추가 | 소 |
| 롱프레스 쓰다듬기 | 누르는 동안 하트 파티클 연속 발생(pet 판정은 기존 1회) | 소 |
| 환경음 | 비/밤 루프 사운드(Sound 모듈 확장, 설정 토글) | 중 (사운드 자산) |
| 단짝 연출 | 11차 사회 행동 데이터로 자주 어울린 쌍에 💕 표시 강화 | 소 |
