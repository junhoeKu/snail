# 아트 스타일 가이드 — Snail 리스타일 공통 규칙

> 모든 캐릭터/배경 이미지는 이 문서의 규칙을 따라 생성한다.
> 그림체 방향: **동화책 일러스트풍(수채+과슈)** — 3차 실사풍 정체성의 연장선. 카툰처럼 유치하지 않고, 사진처럼 차갑지 않게.

---

## 1. 공통 그림체 (모든 프롬프트에 이미 포함되어 있음)

- 부드러운 **과슈/수채 텍스처**, 은은한 종이 질감
- **채도 낮춘 자연 톤** (현재 UI 팔레트와 충돌하지 않게: 이끼 초록 `#74905f`, 흙 갈색 `#857053` 계열)
- 조명은 **왼쪽 위에서 부드럽게**, 강한 콘트라스트 금지
- 색연필 느낌의 얇고 어두운 외곽선 (완전한 무외곽선 금지 — 게임 배경 위에서 실루엣이 살아야 함)
- 텍스트/워터마크/서명 절대 금지

## 2. 캐릭터 공통 기술 규격

| 항목 | 값 | 이유 |
|------|-----|------|
| 크기 | 1024×1024 | 다운스케일 여유 |
| 포맷 | **PNG, 투명 배경** | 서식지 위에 합성 |
| 방향 | **몸통·머리가 왼쪽을 향함** (side view) | 게임의 좌우 반전 로직 기준 (`FLIP_RIGHT: -1`) |
| 구도 | 전신, 중앙 배치, 캔버스 폭의 ~80% | 잘림 방지 |
| 바닥 그림자 | **없음** | CSS `drop-shadow`로 처리 |
| 배경 요소 | **없음** (풀 한 포기도 금지) | 스프라이트 순수성 |

## 3. 생성 순서 & 일관성 유지 요령

1. 변이마다 **성체를 가장 먼저** 생성한다 (그 변이의 기준 이미지)
2. 아기/어린은 성체 이미지를 **참조 이미지로 첨부**하고 "same character, same art style" + 해당 스테이지 프롬프트로 생성한다
3. 첫 변이(갈색)의 성체가 전체 시리즈의 기준이다 — 다른 변이 생성 시에도 참조로 첨부하면 그림체가 흔들리지 않는다
4. 마음에 드는 결과가 나오면 **시드/이미지를 보관**하고, 재생성 시 항상 참조로 사용한다

## 4. 성장 단계 공통 비례 (모든 변이 동일)

| 단계 | 비례 | 분위기 |
|------|------|--------|
| 아기 (baby) | 머리·눈이 크고 껍질은 몸의 ~40%, 전체적으로 동글동글 | 호기심, 연약함 |
| 어린 (junior) | 껍질이 몸의 ~55%로 커지고 나선 뚜렷 | 씩씩함 |
| 성체 (adult) | 껍질 ~65%, 색 깊고 풍부, 자세 당당 | 의젓함 |

## 5. 표정 모디파이어 (선택 — 프롬프트 끝에 한 줄 추가)

게임의 컨디션 시스템과 대응한다. 1차로는 기본 포즈만 생성하고, 여유가 되면 성체부터 표정 세트를 추가한다.

| 인게임 상태 | 추가할 문구 |
|-------------|------------|
| 행복 (`cond-happy`) | `, cheerful expression, eye stalks perked up happily, tiny smile` |
| 배고픔 (`cond-hungry`) | `, tired droopy expression, eye stalks drooping downward, slightly deflated posture` |
| 낮잠 (`state-napping`) | `, sleeping peacefully, eyes closed, curled up snugly against its shell` |
| 먹는 중 (`state-eating`) | `, happily munching on a small lettuce leaf, cheeks slightly puffed` |

## 6. 파일 배치 & 통합 계획

```
assets/
├── characters/
│   ├── egg.png
│   └── snail_{variant}_{stage}.png   ← 예: snail_brown_adult.png (5변이 × 3단계 = 15장 + 알 1장)
└── backgrounds/
    └── bg_{theme}.png                ← bg_moss.png, bg_garden.png
```

- 통합 시 인라인 SVG를 `<img>` 스프라이트로 교체한다 (`feat/restyle` 브랜치, 3계층 구조·상태 머신은 그대로)
- 변이 색상을 CSS 변수로 바꾸던 방식은 **변이별 개별 이미지**로 대체된다
- 컨디션 표정은 이미지 세트가 없는 동안 기존 CSS 변형을 유지한다

## 7. 소품 (참고)

상추/장식(조약돌·버섯·들꽃·이끼바위)도 같은 스타일 블록으로 생성하면 된다. 5차 `feat/decorations` 진행 시 이 가이드에 프롬프트를 추가한다.
