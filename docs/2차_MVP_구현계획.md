# 2차 MVP 구현 계획 — 동적 서식지 (살아있는 달팽이)

> 작성일: 2026-07-11
> 선행 조건: 1차 MVP 완료 ([1차_MVP_구현계획.md](1차_MVP_구현계획.md))
> 검증 가설: **"달팽이가 '살아있는 것처럼' 느껴지면 더 자주, 더 오래 들여다보는가?"**

---

## 1. 개요

### 1.1 무엇이 바뀌는가

1차 MVP의 달팽이는 카드 중앙에 고정되어 `snail-bob` CSS 애니메이션만 반복하는 **정적인 육성 화면**이다.
2차에서는 홈 카드 안에 **서식지(habitat)** 영역을 만들고, 달팽이가 실제 좌표를 가지고 돌아다니며, 사용자가 서식지를 터치하면 그 위치에 상추가 떨어지고 달팽이가 다가가 먹는 **펫 시뮬레이션**으로 바꾼다.

```
[기존]  버튼 클릭 → 스탯 숫자 변화          (즉시, 추상적)
[2차]   터치 → 상추 낙하 → 달팽이 이동 → 먹기 → 스탯 변화   (과정이 보임)
```

### 1.2 바뀌지 않는 것

- 게임 규칙과 수치는 1차 spec §3 그대로다. 먹이 효과(배고픔 -30, 경험치 +10, 행복 +5, 코인 +2)와 경제 밸런스를 **변경하지 않는다**.
- 산책/접속 보상/상점/배경/저장/성장 단계 등 기존 기능은 전부 유지된다. 2차는 "먹이 주기의 표현"을 바꾸는 작업이다.
- 기술 스택 제약(Vanilla JS + LocalStorage, 프레임워크·게임엔진 금지)도 그대로다.

### 1.3 원안 대비 결정 (취사선택 기록)

| 원안 내용 | 결정 | 이유 |
|-----------|------|------|
| 서식지 + 좌표 이동 + 상태 머신 + rAF 루프 | ✅ 가져감 | 2차의 핵심 |
| entity/motion/sprite 3계층 분리 | ✅ 가져감 | 이동·들썩임·반전 transform 충돌 방지에 필요 |
| 먹이 드롭 단일 함수, 동시 1개 제한 | ✅ 가져감 | 단순하고 안전 |
| 탭 비활성화 시 루프 정지, deltaTime 보정 | ✅ 가져감 | 배터리/프레임 독립성 |
| `FOOD_EFFECTS` 별도 상수 (hunger 15/exp 5 등) | ❌ 버림 | 기존 `GAME.CONFIG`와 수치 이원화. 먹이 효과는 `GAME.feed()` 그대로 사용 |
| `snail-controller.js`/`food-controller.js`/`game-loop.js` 3파일 분리 | 🔧 디벨롭 | 이 규모에 3파일은 과함(원안도 인정). **`habitat.js` 단일 IIFE 모듈**로 통합, 내부에서 책임별 함수로 구획 |
| 먹이가 이미 있으면 "옮기기 또는 토스트" | 🔧 디벨롭 | 토스트로 통일 (옮기기는 seeking 중 상태 꼬임 유발) |
| 위치를 px로 저장 | 🔧 디벨롭 | **0~1 비율 좌표**로 저장해 화면 크기 변화에 안전하게 |
| 예시 코드의 speed 18 등 하드코딩 수치 | 🔧 디벨롭 | 모션 수치는 `HabitatModule` 상단 `MOTION` 상수로 일원화 |

## 2. 범위

### 2.1 포함

| # | 기능 | 내용 |
|---|------|------|
| 1 | 서식지 영역 | 홈 카드 내 280~320px 영역. 달팽이·먹이는 absolute 배치, `overflow: hidden` |
| 2 | 상태 머신 | `idle → wandering → seeking_food → eating → idle` |
| 3 | 자율 배회 | 입력이 없어도 2~5초 쉬었다가 무작위 지점으로 천천히 이동 |
| 4 | 먹이 드롭 | 서식지 터치/클릭 또는 [먹이주기] 버튼 → 상추 낙하 → 달팽이가 다가가 먹음 |
| 5 | 연출 | 좌우 반전, 들썩임, 낙하 바운스, 먹기, 상추 축소·소멸, `+EXP` 플로팅 텍스트 |
| 6 | 위치 저장 | 마지막 위치(비율 좌표)를 저장, 재실행 시 이어짐 |

### 2.2 제외 (2차에서 하지 말 것)

- ❌ 먹이 동시 여러 개 / 먹이 종류 추가
- ❌ 장애물, 경로 탐색(패스파인딩), 물리 엔진
- ❌ 달팽이 다중 개체
- ❌ 서식지 꾸미기 아이템 (배경 테마 연동만)
- ❌ 게임 수치/경제 밸런스 변경
- ❌ Canvas/WebGL 도입 — DOM + CSS transform으로만 구현

## 3. 서식지 구조

```html
<div class="snail-habitat" id="snail-habitat">
  <div class="snail-entity" id="snail-entity">   <!-- 좌표 이동 (translate3d) -->
    <div class="snail-motion">                   <!-- 들썩임/먹기 애니메이션 -->
      <span class="snail-sprite">🐌</span>       <!-- 크기(성장 단계)/좌우 반전 -->
    </div>
  </div>
  <div id="food-layer"></div>                    <!-- 상추 -->
</div>
```

- `position: relative`, 높이 280~320px, 모바일 반응형, `overflow: hidden`
- 위치 이동은 `left/top` 대신 `transform: translate3d()` 사용
- 배경은 기존 테마 변수 재사용 + 잔디 느낌의 최소 장식. body 배경 테마(풀밭/정원)와 톤을 맞춘다
- 기존 카드/스탯 바/행동 버튼 디자인은 유지하고, 스프라이트의 `stage-*` 클래스(성장 단계별 크기·색)도 그대로 재사용한다
- 알(`egg`) 단계에서는 서식지를 표시하지 않는다 — 기존 egg-view 온보딩 유지

## 4. 상태 머신 & 모션

### 4.1 상태 정의

| 상태 | 동작 | 전이 |
|------|------|------|
| `idle` | 정지 + 들썩임 애니메이션 | 2~5초(랜덤) 후 → `wandering`. 먹이 생성 시 즉시 → `seeking_food` |
| `wandering` | 무작위 목표 좌표로 천천히 이동 | 도착 → `idle`. 먹이 생성 시 즉시 → `seeking_food` |
| `seeking_food` | 상추 좌표로 이동 (배회보다 약간 빠름) | 상추와 거리 ≤ 30px → `eating` |
| `eating` | 정지 + 먹기 애니메이션, 상추 점점 축소 | 1.8초 후 상추 제거 + `GAME.feed()` 반영 → `idle` |

### 4.2 모션 수치 (`HabitatModule`의 `MOTION` 상수로 일원화)

> `GAME.CONFIG`는 게임 밸런스 수치 전용으로 유지한다. 연출·모션 수치는 `habitat.js` 상단 `MOTION`에만 둔다.

| 항목 | 값 |
|------|-----|
| 배회 속도 | 18 px/s |
| 먹이 추적 속도 | 28 px/s |
| idle 지속 | 2000~5000ms 랜덤 |
| 먹기 지속 | 1800ms |
| 먹기 판정 거리 | 30px |
| 도착 판정 거리 | 2px |

### 4.3 이동 로직

- `requestAnimationFrame` 게임 루프 + `deltaTime` 보정 (프레임 수와 무관하게 등속)
- 이동 방향에 따라 스프라이트 좌우 반전: 오른쪽 `scaleX(1)` / 왼쪽 `scaleX(-1)` — 반전은 `.snail-sprite`, 좌표는 `.snail-entity`에 적용해 transform 충돌을 피한다
- 좌표는 항상 서식지 경계 안으로 클램프. 성장 단계별 스프라이트 크기를 감안해 여백을 둔다
- `document.visibilitychange`로 탭 비활성화 시 루프 일시 정지, 복귀 시 재개 (deltaTime 점프 방지)
- 상점 화면으로 전환하면 루프를 멈추고, 홈으로 돌아오면 재개한다

## 5. 먹이 드롭

### 5.1 흐름

```
터치/버튼 → 사전 검증 → 상추 낙하 연출 → 달팽이 추적 → 먹기 → GAME.feed() 정산
```

- 진입점은 두 가지, 로직은 `dropFood(x, y)` 단일 함수:
  - 서식지 `pointerdown` (클릭+터치 통합) → 해당 좌표
  - 기존 [먹이주기] 버튼 → 서식지 내 임의 좌표
- 클릭 좌표는 서식지 내부 좌표로 변환하고 가장자리에서 잘리지 않게 보정한다
- **동시에 1개만** 존재. 이미 있으면 "달팽이가 먹고 있어요!" 토스트만 표시 (중복 생성/이동 없음)
- 낙하 시 위에서 떨어져 통통 튀는 애니메이션

### 5.2 재고·효과 정산 시점 (중요)

- **드롭 시점**: 사전 검증만 한다 — 알 단계 아님, 상추 보유(`player.food ≥ 1`), 배고픔 > 0. 실패하면 기존 실패 토스트(no_food/not_hungry) 재사용
- **먹기 완료 시점**: `GAME.feed(snail, player)`를 호출해 상추 소모와 효과(배고픔/경험치/행복/코인)를 **한 번에** 정산한다. 수치를 새로 정의하지 않는다
- 먹는 도중 앱을 종료하면 진행 중이던 먹이만 사라진다 (상추는 소모되지 않았으므로 손해 없음 — `eating` 상태는 저장하지 않는다)
- levelup/stage_up 이벤트 연출은 기존 `HomeModule._showEvents` 경로를 재사용한다

## 6. 데이터 모델 변경

| 키 | 변경 | 내용 |
|------|------|------|
| `sn_snail` | 필드 추가 | `pos: { rx, ry }` — 서식지 내 **0~1 비율 좌표** (px 아님). `schema_version: 2` |

- 비율 좌표라 화면/서식지 크기가 바뀌어도 안전하다. 렌더 시 px로 환산한다
- 기존 사용자(v1 레코드)는 `DB._getOrInit`의 기본값 병합으로 자동 치유된다 — `pos`가 없으면 중앙(`{rx: 0.5, ry: 0.5}`)에서 시작
- 이동 목표(target)와 상태 머신 상태는 **저장하지 않는다** (재실행 시 항상 `idle`부터)

## 7. 모듈 구조

```
js/habitat.js  ← ★ 신규. 전역 네임스페이스: HabitatModule (IIFE)
```

내부 책임 구획 (파일 분리 대신 함수·주석으로 구분):

```
HabitatModule
├─ MOTION                  ← 모션 수치 상수 (§4.2)
├─ 상태 머신               ← setState / update별 핸들러
├─ 게임 루프               ← rAF + deltaTime, pause() / resume()
├─ 먹이 관리               ← dropFood(x, y) / removeFood()
└─ 렌더                    ← translate3d 반영, 반전, 연출 클래스 토글
```

기존 모듈과의 경계:

| 모듈 | 역할 분담 |
|------|-----------|
| `GAME` | 변경 없음 (순수 함수 유지). 먹기 정산은 기존 `GAME.feed()` 호출 |
| `DB` | `sn_snail` 기본값에 `pos` 추가, `schema_version` 2 |
| `HomeModule` | 스탯 바/이벤트 토스트 렌더 유지. [먹이주기] 버튼이 `HabitatModule.dropFood()` 위임으로 변경 |
| `App` | navigate 시 `HabitatModule.pause()/resume()` 호출 |
| `css/home.css` | 서식지/먹이/모션 애니메이션 스타일 추가 |

**금지**: `habitat.js`에서 `localStorage` 직접 접근, `GAME` 외부에서 스탯 수치 직접 가감, 먹이 효과 수치 재정의.

## 8. 예외 처리 체크리스트

- 알 단계에서는 게임 루프를 시작하지 않는다
- 성장 단계별 크기(56~104px)가 달라도 경계를 넘지 않는다 (크기 기반 클램프)
- 빠른 연속 터치에도 먹이가 중복 생성되지 않는다 (1개 제한이 자연 방어)
- `eating` 중 터치해도 상태가 꼬이지 않는다 (eating 중 드롭 거부)
- 리사이즈 시 좌표를 안전 범위로 보정한다 (비율 좌표라 대부분 자동 해결)
- 상점 화면에서 루프가 돌지 않고, 홈 복귀 시 정상 재개된다
- 서식지에 `touch-action` 처리로 모바일 스크롤과의 충돌을 완화한다

## 10. 완료 조건

1. 입력이 없어도 달팽이가 서식지 안을 가끔 천천히 돌아다닌다
2. 서식지를 터치하면 해당 위치에 상추가 떨어진다
3. 달팽이가 상추까지 자연스럽게 이동한다 (배회보다 빠르게, 방향 반전 정상)
4. 도착하면 먹기 애니메이션이 실행되고 상추가 점점 작아진다
5. 먹은 뒤 상추가 제거되고 스탯이 **기존 수치대로** 갱신된다 (배고픔 -30, EXP +10, 행복 +5, 코인 +2, 상추 -1)
6. 달팽이가 어떤 성장 단계에서도 서식지 밖으로 나가지 않는다
7. 모바일에서 터치, 데스크톱에서 클릭이 모두 동작한다
8. 기존 성장/상점/배경/저장/접속 보상이 깨지지 않는다
9. 콘솔 에러 0개, 상태 전환이 `idle → wandering → seeking_food → eating → idle` 순환으로 정상 동작한다
10. 재실행 시 마지막 위치(또는 중앙)에서 시작하고, 1차 데이터(v1)를 가진 사용자도 오류 없이 이어진다

### 검증 방법

```js
// 브라우저 DevTools Console
DB.Snail.get().pos;              // { rx, ry } 저장 확인
HabitatModule.debugState();      // 현재 상태 머신 상태 확인 (디버그용 export)
```

- jsdom 통합 테스트에 서식지 부팅/드롭 검증을 추가한다 (rAF은 jsdom `pretendToBeVisual`로 동작)
- 실기기(모바일 390px)에서 터치 드롭과 스크롤 충돌을 육안 확인한다
