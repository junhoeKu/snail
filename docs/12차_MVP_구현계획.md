# 12차 MVP 구현 계획 — 탭 개편 · 설정 분리 · 도감 등급 · 미니게임 전환

> 작성일: 2026-07-13
> 선행 조건: 11차(살아있는 생태계·슬롯 8마리) 완료 — v1.3.0 배포됨
> 목표 기간: 2~3주
> 검증 가설: **"정보 구조가 정리되고 각 화면이 명확해지면, 유저가 원하는 걸 더 쉽게 찾고 오래 머무는가?"**
> 성격: 신규 콘텐츠보다 **정보 구조(IA)·UX 정리** 중심. 미니게임은 그릇만 만들고 콘텐츠는 후속(13차 인접).

## 1. 부화/성장 아이콘 = 해당 달팽이 스프라이트 ✅ (이번 반영 완료)

- **문제**: 부화/성장 축하 모달의 아이콘이 이모지(🐌)라 어떤 색·외형인지 알 수 없었다.
- **해결**: `Toast.celebrate`에 `image` 옵션 추가 → 부화 시 `spritePath(color,'baby')`, 성장 시 `spritePath(color, stage)`를 표시. 서버(`api.js` hatched/stage_up)·로컬(`home.js`) 양쪽 반영. 여행(graduate)은 🧳 유지.
- 파일: `js/toast.js`, `js/api.js`, `js/home.js`, `css/components.css`(`.modal-emoji-img`).

> 이 항목은 12차 착수 전에 선반영했다. 아래 2~5가 12차 본작업이다.

## 2. 탭 개편 — 유저 / 도감 / 홈 / 탐험 / 상점

### 2.1 현재 → 목표
| 위치 | 현재 | 12차 |
|------|------|------|
| 탭1 | 스탯(양육자+도감+앨범+일지) | **유저** (양육자+앨범+일지) |
| 탭2 | 상점 | **도감** (변이 도감, 등급별) |
| 탭3(중앙) | 홈 | 홈 (유지) |
| 탭4 | 탐험 | 탐험 (미니게임 그릇으로 §4) |
| 탭5 | 설정 | **상점** |
| — | — | 설정은 탭에서 빼고 **홈 우측 상단 아이콘**으로 (§3) |

- 기존 `stats.js`(양육자/도감/앨범/일지 한 화면)를 **유저 화면**(양육자·앨범·일지)과 **도감 화면**(변이 도감)으로 분리한다.
- `index.html` 탭바 5개 라벨/아이콘/`data-screen` 교체, 화면 컨테이너 `#screen-dex` 신설.
- `app.js` 라우팅(`navigate`)에 `dex` 추가, 각 탭 진입 시 렌더 호출.

### 2.2 파일
- `index.html`: 탭바 재배열, `#screen-user`/`#screen-dex` 섹션.
- `js/stats.js` → **분리**: `js/user.js`(UserModule: 양육자/앨범/일지), `js/dex.js`(DexModule: 도감). 또는 stats.js를 UserModule로 두고 DexModule만 신설(변경 최소 우선).
- `js/app.js`: 네비 라우팅.

## 3. 설정 — 홈 우측 상단 아이콘 + 하위 화면

### 3.1 배치
- 홈 헤더: **돈·먹이 표시를 살짝 왼쪽으로**, 가장 우측 상단에 **⚙️ 설정 아이콘** 배치.
- 설정 아이콘 클릭 → 설정 화면(오버레이 또는 `#screen-settings` 진입).

### 3.2 설정 항목 (각각 별도 하위 화면)
| 항목 | 내용 |
|------|------|
| 규칙 | 게임 규칙 안내 (기존 rules-list) |
| 배경 | **상점에서 설정으로 이동** — 배경 선택(default/garden…) |
| 사운드 | 소리 켜기/끄기 |
| 데이터 | 백업 코드 내보내기/복구 + 초기화 |

- 각 항목 진입 시 해당 화면만 표시(설정 서브 라우팅). 뒤로가기 = 설정 목록.
- 배경 선택 UI를 `deco.js`/`shop.js`에서 설정으로 이관.

### 3.3 추가하면 좋은 설정 항목 (추천)
1. **계정** — 서버 모드에서 게스트/소셜 연결 상태, 로그아웃/기기 이전(백업 코드 대체·보완)
2. **알림** — PWA 푸시(배고픔/편지 도착) 온오프 — 인프라 후속
3. **다크 모드 / 밤 테마** — 11차 밤낮 오버레이와 연결, 수동 토글
4. **모션 줄이기(접근성)** — 이동/이모트 애니메이션 최소화 옵션
5. **정보** — 버전·크레딧·문의(이메일/폼)·오픈소스 고지
6. **언어** — 향후 다국어 대비(현재 한국어 전용)

> 12차에서는 규칙/배경/사운드/데이터 4개만 구현. 위 추천은 항목별 난이도·의존성(푸시=서버, 계정=인증)을 표기해 13차+로 배분.

## 4. 탐험 → 미니게임 전환

### 4.1 달팽이 찾기(야생 알) 이벤트 제거
- `explore_roll`의 `egg`(야생 알 5%) 결과 제거. 확률 재분배: 코인 55% / 상추 25% / 꽝 20% (합 1). 클라(`game.js`)·서버(`rules.py`) 미러 + 원격 설정 검증(확률 합=1).
- `wildEggVariant`·`convertWildEgg`·`WILD_EGG_FALLBACK_COINS` 및 서버 egg 분기 정리(사용처 제거, 상수는 남겨도 무해하나 주석 정리).
- 슬롯 확장은 상점 구매로만(§11차 §6) — 야생 알 수용 로직 삭제.

### 4.2 탐험 탭 → 미니게임 허브 (그릇만, 콘텐츠 후속)
- 탐험 탭을 **미니게임 목록**으로: 탐험(채집) / 달팽이 경주 / 달팽이 퀴즈.
- 12차에서는 **탐험(채집)만 기존 그대로** 미니게임 목록의 한 항목으로 배치하고, 경주·퀴즈는 "준비 중" 플레이스홀더 카드.
- 경주·퀴즈 실제 구현은 **13차 이후 후속**(별도 게임 루프·문항 데이터 필요).

### 4.3 파일
- `js/game.js`·`rules.py`: `explore_roll` 확률, egg 제거.
- `js/explore.js`: 미니게임 허브 UI(탐험 카드 + 경주/퀴즈 플레이스홀더).
- 테스트: 탐험 확률·야생 알 제거 pytest/jsdom 갱신.

## 5. 도감 등급별 분리

- 도감 화면(§2 DexModule)을 **기본 / 레어 / 에픽** 섹션으로 나눈다(현재는 평면 나열).
- 각 섹션 헤더 + 진행률(예: 기본 8/10), 미발견은 실루엣. 등급 배지는 기존 `rarity-badge` 재사용.
- 현재 변이: 기본 10 + 레어(연못) 1. 에픽 섹션은 비어 있음(히든 변이는 13차) — 섹션은 미리 만들어 "곧 만나요" 표시.
- 파일: `js/dex.js`, `css/stats.css`(섹션 스타일).

## 6. 앨범 내부 스크롤

- 유저 화면(§2)의 앨범 목록이 길어지면 페이지 전체가 늘어남 → **성장 일지처럼 내부 스크롤**.
- `css/stats.css` `.album-list`에 `max-height` + `overflow-y: auto`(일지 `.journal-list`와 동일 패턴).
- 파일: `css/stats.css`.

## 8. 완료 조건

1. 부화/성장 시 해당 달팽이 스프라이트가 축하 모달에 뜬다 ✅(선반영)
2. 탭이 유저/도감/홈/탐험/상점 5개로 재구성되고 각 화면이 분리 렌더된다
3. 설정이 홈 우측 상단 아이콘으로 들어가고, 규칙/배경/사운드/데이터 하위 화면이 동작한다(배경은 상점→설정 이관)
4. 탐험에서 달팽이 찾기(야생 알)가 사라지고, 탐험 탭이 미니게임 허브(탐험+경주/퀴즈 플레이스홀더)가 된다
5. 도감이 기본/레어/에픽 섹션으로 나뉜다
6. 앨범이 내부 스크롤로 바뀐다
7. 기존 기능(성장/경제/서버 판정/생태계) 무회귀, 모바일 390px 깨짐 없음
