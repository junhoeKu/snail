# 6차 MVP 구현 계획 — 멀티 달팽이 & 탐험 채집 & 앱 기반

> 작성일: 2026-07-11
> 선행 조건: 5차 MVP + v0.6.0 튜닝 완료 (산책 제거/쿨다운 제거/관리자 모드)
> 목표 기간: 4~6주 (역대 최대 규모 — 구조 변경 포함)
> 검증 가설: **"여러 마리를 모으고, 직접 채집하러 나가는 재미가 헤비 유저를 만드는가?"**

---

## 1. 개요

### 1.1 방향 전환

6차는 게임의 정체성을 한 단계 확장한다:

```
[~5차] 혼자 키우는 게임        →  [6차] 여럿을 "모으고", 직접 "채집"하는 게임
        (돌봄 루프)                     (돌봄 + 수집 + 탐험 루프)
```

동시에 유저가 실제로 유입되는 시점이므로 **앱 기반(PWA/백업)** 을 함께 다진다.

### 1.2 요구사항 → 설계 매핑

| 요구 | 설계 |
|------|------|
| 먹이 동시 1개 제한 제거 | 서식지에 상추 여러 개 드롭 가능 (필드 상한 10개). 달팽이들이 가까운 먹이부터 나눠 먹음 (§4.1) |
| 달팽이 알 상점 구매 | 상점에서 "달팽이 알(새 보금자리)" 판매 — 구매 = 슬롯 확장 + 알 획득, 최대 3마리 (§4.3) |
| 헤비 유저(현질러) 욕구 | 상추 묶음 구매(×10 할인) + 알/슬롯 구매 + 탐험 입장권 등 **코인 소비처 확충**. 실제 인앱결제는 여전히 범위 밖 — 지금은 "코인을 쓸 곳"의 구조를 만드는 단계 (§1.3) |
| 야외 채집 (포켓몬고식) | GPS 오픈월드는 보류(§1.3), **인게임 탐험 맵** 채집으로 시작: 맵에 들어가 수풀을 뒤져 코인/상추/야생 알 발견 (§4.4) |
| 기존 6차 (기반/바이럴) | PWA + 데이터 백업/복구 포함. 무늬 변이·공유 카드는 7차로 이월 (§1.3) |

### 1.3 트리아지 결정 (근거 기록)

| 항목 | 결정 | 이유 |
|------|------|------|
| GPS 기반 오픈월드 채집 | ⏭ 보류 (7차+ 검토) | 위치 권한 UX, 지도 타일(외부 의존성 — 스택 제약 충돌), 실외 테스트 비용. 단, **PWA + Geolocation은 브라우저 내장**이라 "동네 산책 모드"(지도 없이 이동 거리 기반 채집) 형태로는 7차+에서 가능 — 인게임 맵의 채집 루프가 검증되면 그 위에 얹는다 |
| 실제 결제(현질) | ❌ 범위 밖 유지 | 결제는 사업자/스토어/법적 준비가 필요. 6차는 헤비 유저가 코인을 쓰고 싶어지는 **소비 구조**까지만 — 이 구조가 곧 미래 수익화의 설계도가 된다 |
| 무늬 변이 (2차원 도감) | ⏭ 7차 이월 | 멀티 달팽이 + 맵별 변이 가중치(§4.4)가 이미 이번 사이클의 수집 확장이다. 한 사이클에 수집 축을 두 개 늘리면 검증이 흐려진다 |
| 공유 카드 | ⏭ 7차 이월 | 카드의 비주얼 = 바이럴 성능. `feat/restyle`(GPT 이미지) 이후가 효율적 |
| 장식 확장 | ⏭ 7차 이월 | 꾸미기 탭이 탐험 탭으로 대체되므로(§6.1) 기존 장식은 상점으로 이동만 하고 확장은 보류 |

## 2. 범위

### 2.1 포함 (기능 6개)

| # | 기능 | 내용 |
|---|------|------|
| 1 | **멀티 달팽이** | 최대 3마리 동시 사육. 각자 독립된 스탯/성격/변이/성장/여행. 서식지에서 함께 배회 |
| 2 | **먹이 다중 드롭** | 동시 여러 개(상한 10). 배고픈 달팽이들이 가까운 상추부터 각자 찾아가 먹음 |
| 3 | **상점 개편** | 상추 ×1/×10 묶음, 달팽이 알(슬롯 확장), 배경/장식 통합 (꾸미기 탭 흡수) |
| 4 | **탐험 채집** | 인게임 맵 2+1곳 입장 → 수풀/돌 뒤지기(하루 스태미나) → 코인/상추/야생 알 발견. 맵별 변이 가중치 |
| 5 | **PWA** | 홈 화면 설치 + 오프라인 캐시 + 앱 아이콘. iOS 7일 스토리지 삭제 리스크 완화 |
| 6 | **데이터 백업/복구** | 설정 탭에서 내보내기(복사 가능한 코드) / 가져오기(확인 모달 필수) |

### 2.2 제외

- ❌ GPS/지도/실외 위치 기반 플레이 (7차+ 검토)
- ❌ 실제 결제/광고
- ❌ 무늬 변이, 공유 카드, 장식 신규 (7차)
- ❌ 달팽이 4마리 이상, 교배
- ❌ 서버/계정/멀티플레이

## 3. 데이터 모델 (`schema_version: 5`)

| 키 | 변경 | 내용 |
|------|------|------|
| `sn_snails` | **신규 (구조 변경)** | 달팽이 배열 `[{ id, name, level, ..., pos, personality, color }]`. 각 레코드는 기존 `sn_snail` 구조 + `id` |
| `sn_snail` | 마이그레이션 후 삭제 | 부팅 시 기존 1마리에 `id` 부여 → `sn_snails[0]`로 이전 |
| `sn_player` | 필드 추가 | `snail_slots`(기본 1, 최대 3), `explore: { date, searches }`(스태미나), `unlocked_maps: []` |
| `sn_album`, `sn_journal` | 유지 | 일지/앨범 문구에 달팽이 이름 포함 (이미 그렇게 동작) |

- **마이그레이션 필수**: 기존 유저의 `sn_snail` → `sn_snails` 자동 이전 (1회, 데이터 손실 없음). QA에서 최우선 검증
- 알도 배열의 한 원소 (`stage: 'egg'`) — 알 여러 개 공존 가능 (구매 직후 + 여행 직후 등)

## 4. 게임 규칙 (수치 — `GAME.CONFIG` 추가/변경)

### 4.1 먹이 다중화

| 항목 | 값 |
|------|-----|
| 필드 동시 상추 상한 | 10개 (성능/UX 가드 — "제한 없음"의 실질 구현) |
| 드롭 검증 | 상추 재고 1개 이상 && 배고픈(hunger>0) 달팽이가 1마리 이상 |
| 분배 | 배고픈 달팽이 중 **가장 가까운** 개체가 그 상추를 점유(claim). 한 마리 = 동시에 1개만 추적 |
| 정산 | 기존 `GAME.feed()` 그대로 — 먹은 달팽이에게만 적용 |

### 4.2 멀티 달팽이

| 항목 | 값 |
|------|-----|
| 최대 사육 | 3마리 (슬롯제) |
| 시간 감쇠/부재 정산 | 각 개체 독립 적용 |
| 여행 보내기 | 개체별 (성체 Lv12+ 동일). 여행 후 그 슬롯에 새 알 무료 도착 (기존 루프 유지) |
| 접속 보상/스트릭/미션 | 계정 단위 유지 (미션 "먹이 2회"는 아무 달팽이나 합산) |
| 쓰다듬기 | 터치한 개체에게 적용 |

### 4.3 상점 개편

| 상품 | 가격 | 비고 |
|------|------|------|
| 상추 ×1 | 10코인 | 기존 |
| 상추 ×10 | 90코인 | 10% 할인 — 대량 구매 욕구 대응 |
| 달팽이 알 (2번째 보금자리) | 500코인 | 슬롯 2 개설 + 알 즉시 도착 |
| 달팽이 알 (3번째 보금자리) | 1,500코인 | 슬롯 3 개설 + 알 즉시 도착 |
| 배경/장식 | 기존 가격 | 꾸미기 탭에서 이동 |

### 4.4 탐험 채집

| 항목 | 값 |
|------|-----|
| 맵 | 🌿 이끼 숲(기본) · 🌤️ 햇살 들판(기본) · 💧 이슬 연못(**해금**: 2세대 도달 또는 1,000코인) |
| 스태미나 | 뒤지기 10회/일 (로컬 자정 리셋, 맵 공용) |
| 뒤지기 결과 | 코인 55% (3~12) · 상추 25% (1~2) · 꽝 15% (재미 문구: "이슬만 반짝였어요") · **야생 알 5%** |
| 야생 알 | 빈 슬롯이 있어야 획득 (없으면 코인 30으로 자동 전환 + 슬롯 구매 안내). 맵별 변이 가중치: 이끼 숲=올리브↑, 햇살 들판=적갈↑, 이슬 연못=회갈↑·**황금 2배** |
| 데일리 미션 | "탐험 다녀오기 1회" 추가 → 미션 3종 복원 (먹이 2회 · 쓰다듬기 1회 · 탐험 1회) |

- 탐험은 **순수 함수** `GAME.explore(player, mapId, rng)`로 판정 (rng 주입 — 테스트 가능)
- 맵 화면: 전체 화면 일러스트 배경 + 뒤질 수 있는 오브젝트(수풀/돌/낙엽) 6~8개, 터치하면 결과 연출. 오브젝트는 입장마다 위치 셔플

### 4.5 PWA / 백업

| 항목 | 내용 |
|------|------|
| PWA | `manifest.json`(이름/아이콘/standalone) + `sw.js`(정적 자산 캐시 우선, 버전 키로 갱신). 아이콘은 docs/art 스타일로 생성 (프롬프트 추가) |
| 백업 | 저장소 전체(JSON) → Base64 코드 문자열 내보내기(클립보드 복사). 가져오기는 코드 붙여넣기 + **확인 모달**(기존 데이터 덮어씀 경고) 후 적용·새로고침 |

## 5. 도메인 로직 (`game.js` — 순수 함수 유지)

| 함수 | 역할 |
|------|------|
| `GAME.feed/pet/gainExp/...` | **시그니처 유지** — 개체 1마리 단위 연산이므로 멀티는 호출부(모듈)가 처리 |
| `GAME.summarizeAway(snails, player, ...)` | 배열 입력으로 확장 — 개체별 감쇠 + 계정 발견 통합 리포트 |
| `GAME.buyFoodBundle(player, count)` | 상추 묶음 구매 (할인 적용) |
| `GAME.buyEggSlot(player)` | 슬롯 개설 + 알 레코드 생성 → `{ player, egg, events }` |
| `GAME.explore(player, mapId, rng)` | 스태미나 차감 + 결과 판정 → `{ player, result, events }` |
| `GAME.wildEggVariant(mapId, generation, rng)` | 맵 가중치 + 세대 보정 변이 추첨 |
| `GAME.exportData(...)` / `GAME.importData(code)` | 백업 코드 인코딩/디코딩 + 유효성 검증 (스키마 확인) |

## 6. 화면/모듈 변경

### 6.1 탭 개편

```
[~5차] 달팽이 · 상점 · 🏠홈 · 꾸미기 · 설정
[6차]  달팽이 · 상점 · 🏠홈 · 🧭탐험 · 설정     ← 꾸미기는 상점으로 흡수
```

### 6.2 화면별

| 화면 | 변경 |
|------|------|
| 홈 | 여러 마리 동시 배회 + 알은 서식지 오브젝트로 표시(터치 → 이름 짓기 모달). 좌상단 칩은 "달팽이 3마리 · 배고픈 아이 1" 요약 → 탭하면 달팽이 탭. 기존 전체 화면 알 온보딩은 첫 실행에만 유지 |
| 달팽이 탭 | 상단에 개체 선택 칩(이름별) → 아래 기존 상세(스탯/특성/여행/일지). 도감/앨범은 계정 공통 섹션 유지 |
| 상점 | 상추 ×1/×10 · 달팽이 알 · 배경 · 장식 (섹션 구분) |
| 탐험 (신규) | 맵 선택 카드(스태미나 표시) → 맵 화면(오브젝트 터치 채집, 결과 토스트/모달) |
| 설정 | 백업/복구 섹션 추가 |
| 신규 모듈 | `explore.js`(ExploreModule), `backup.js`(BackupModule 또는 SettingsModule 내 구획) |

| 2 | `feat/multi-habitat` | HabitatModule 엔티티 배열 리팩터링 + 먹이 다중 드롭/점유 + 개체별 쓰다듬기 | 3마리 동시 배회·급식 정상 |
| 3 | `feat/multi-ui` | 달팽이 탭 개체 선택 + 홈 요약 칩 + 알 오브젝트/이름 짓기 + 개체별 여행 | 여행 → 새 알 → 부화 루프 개체별 동작 |
| 4 | `feat/shop-v2` | 상추 묶음 + 알/슬롯 판매 + 꾸미기 흡수 (탭 개편 1/2) | 슬롯 2·3 구매 → 알 도착 |

| 6 | `feat/pwa` | manifest + service worker + 아이콘 (아트 프롬프트 추가) | 설치 가능 + 오프라인 부팅 |
| 7 | `feat/backup` | 내보내기/가져오기 (확인 모달) | 코드 왕복 후 데이터 동일 |
| 8 | `chore/v6-qa` | 통합 테스트 전면 갱신(멀티/탐험/마이그레이션), 실기기 QA, v0.7.0 | §8 전체 통과 |

> 1~3단계(멀티 달팽이)가 이번 사이클의 리스크 덩어리다. 4단계 이후는 독립적이므로, 멀티가 늦어지면 PWA/백업(6·7)을 먼저 빼서 배포해도 된다.

## 8. 완료 조건

1. 기존 유저가 업데이트 후 접속해도 달팽이/코인/앨범이 그대로다 (단일→배열 마이그레이션)
2. 상추를 연속으로 여러 개 던질 수 있고, 배고픈 달팽이들이 각자 가까운 상추를 찾아가 먹는다
3. 상점에서 알을 사면 새 보금자리에 알이 도착하고, 부화시켜 3마리까지 동시에 키울 수 있다
4. 각 달팽이가 독립적으로 성장·여행하고, 여행 후 그 자리에 새 알이 온다
5. 탐험 맵에서 하루 10회 뒤지기로 코인/상추/야생 알을 얻고, 야생 알은 맵별 변이 가중치를 따른다 (시드 고정 검증)
6. 슬롯이 가득할 때 야생 알은 코인으로 전환되고 안내가 나온다
7. 홈 화면에 앱으로 설치되고, 비행기 모드에서도 실행된다
8. 백업 코드로 다른 브라우저에서 완전히 같은 상태를 복원할 수 있다
