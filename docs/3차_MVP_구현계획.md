# 3차 MVP 구현 계획 — 탭 레이아웃 & 실사풍 비주얼

> 작성일: 2026-07-11
> 선행 조건: 2차 MVP 완료 ([2차_MVP_구현계획.md](2차_MVP_구현계획.md))
> 검증 가설: **"서식지가 넓어지고 게임틱한 느낌이 빠지면 더 오래 머물고 더 자주 열어보는가?"**

---

## 1. 개요

### 1.1 무엇이 바뀌는가

3차는 **게임 기능을 추가하지 않는다**. 화면 구조와 비주얼만 개편한다.

| 항목 | 현재 (2차) | 3차 |
|------|-----------|-----|
| 화면 구조 | 단일 세로 스크롤 (카드 안에 서식지 300px) | **상단 콘텐츠 + 하단 5탭 바** 2섹션 분할 |
| 서식지 크기 | 카드 내부 300px — 좁음 | **상단 영역 전체** (화면 높이의 ~85%) |
| 화면 전환 | 홈 ↔ 상점 버튼 이동 | 하단 탭 바(중앙 홈 버튼)로 5개 화면 전환 |
| 비주얼 | 이모지 + 원색 그라데이션 (게임틱) | **인라인 SVG 달팽이 + 자연 톤 실사풍 배경** |

### 1.2 요구사항 → 설계 매핑

| 요구사항 | 설계 |
|----------|------|
| 화면 두 섹션 분할, 아래 5개 위젯으로 기능 전환 | 하단 고정 탭 바 (§3). 상단은 선택된 탭의 화면 |
| 가운데는 홈 버튼 | 중앙 탭 = 홈. 다른 탭보다 크고 떠 있는 원형 버튼으로 강조 |
| 홈 화면 넓게, 달팽이 돌아다니는 컨텐츠 | 서식지를 상단 영역 전체로 확장. 스탯 바 등은 별도 탭으로 이동, 홈엔 최소 오버레이만 (§4) |
| 게임같지 않게, 실사 느낌 | 이모지 스프라이트 → 셰이딩 있는 SVG 달팽이. 배경은 채도 낮춘 자연 톤 + 텍스처 (§5) |

## 2. 범위

### 2.1 포함

1. 하단 5탭 바: `스탯 · 상점 · 홈(중앙) · 꾸미기 · 설정`
2. 서식지 전체 화면화 + 홈 오버레이(지갑/요약 칩/행동 버튼) 재배치
3. 스탯 화면 분리 (경험치/배고픔/행복 바 + 성장 단계 안내)
4. 꾸미기 화면 분리 (배경 선택 — 상점에서 이동)
5. 설정 화면 신설 (데이터 초기화[확인 모달 필수], 앱 정보, 게임 규칙 안내)
6. SVG 달팽이 캐릭터 + 실사풍 배경 2종 리디자인

### 2.2 제외 (3차에서 하지 말 것)

- ❌ 새 게임 기능/수치 변경 — `GAME`, `DB` 스키마, 경제 밸런스 전부 그대로
- ❌ 외부 이미지/포토 에셋, 이미지 CDN — 에셋은 인라인 SVG와 CSS만
- ❌ Canvas/WebGL 전환 — DOM + transform 구조 유지
- ❌ 달팽이 스킨/커스터마이즈 (꾸미기는 배경 선택만)
- ❌ 다크 모드, 시간대별 배경 변화
- ❌ 탭별 URL 라우팅(해시 등) — 기존 `App.navigate` 방식 유지

### 2.3 기술 제약 갱신

- 2차까지 "달팽이 그래픽은 이모지/CSS"였다. 3차부터 **index.html에 직접 인라인한 SVG**를 허용한다.
  - 외부 파일 fetch 없음(file:// CORS 문제 회피), 빌드 없음, 의존성 없음 — 스택 제약과 충돌하지 않는다.
- Font Awesome(CDN)은 탭 바 아이콘에 계속 사용한다.

## 3. 레이아웃 구조

```
┌──────────────────────────────┐
│  상단 콘텐츠 영역 (flex: 1)     │
│                              │
│   [홈] 서식지 전체 화면          │
│     · 우상단: 지갑 칩 🪙 🥬     │
│     · 좌상단: 이름·Lv·상태 칩    │
│     · 우하단: 먹이/산책 FAB     │
│                              │
├──────────────────────────────┤
│  📊      🛒    ╭────╮   🎨    ⚙️
│  스탯    상점   │ 🏠 │  꾸미기  설정
│               ╰────╯ ← 중앙, 크고 떠 있음
└──────────────────────────────┘
```

- `#app`: `height: 100dvh` flex column. 상단 `main`(flex 1, 탭별 화면) + 하단 `nav.tab-bar`(고정 ~64px + safe-area 패딩)
- 기존 상단 헤더(앱 타이틀 + 지갑)는 제거하고, 지갑은 홈 서식지 위 오버레이 칩으로 이동. 다른 탭에서는 각 화면 상단에 표시
- 탭 바는 모든 화면에서 고정 노출. 활성 탭 하이라이트
- 중앙 홈 버튼: 지름 ~56px 원형, 탭 바 위로 살짝 돌출(`translateY(-12px)`), 그림자

### 3.1 화면(탭) 구성

| 탭 | 화면 ID | 모듈 | 내용 |
|----|---------|------|------|
| 📊 스탯 | `screen-stats` | `StatsModule` (신규) | 스탯 바 3종(기존 홈에서 이동), 성장 단계 진행도, 다음 레벨 안내 |
| 🛒 상점 | `screen-shop` | `ShopModule` | 상추 구매만 (배경 선택은 꾸미기로 이동) |
| 🏠 홈 (중앙) | `screen-home` | `HomeModule` + `HabitatModule` | 전체 화면 서식지 + 오버레이 |
| 🎨 꾸미기 | `screen-deco` | `DecoModule` (신규) | 배경 선택 (상점에서 이동), 미리보기 |
| ⚙️ 설정 | `screen-settings` | `SettingsModule` (신규) | 게임 규칙 안내, 데이터 초기화(2단계 확인 모달), 버전 정보 |

- 온보딩(알/이름 짓기)은 홈 화면 안에서 기존대로. 알 단계에서 다른 탭 이동은 허용
- 부팅 시 항상 홈 탭에서 시작 (탭 상태 저장 안 함)
- `App.navigate`가 5개 화면을 처리하고, 홈 이탈 시 `HabitatModule.pause()` / 복귀 시 `resume()` — 기존 로직 유지

## 4. 홈(서식지) 화면

- 서식지가 상단 영역 **전체**를 차지한다 (`height: 100%`, 카드/테두리 제거 — 배경이 곧 화면)
- 위치는 이미 0~1 비율 좌표라 **마이그레이션 불필요** — 영역이 커지면 자동으로 넓게 분포
- 오버레이 (서식지 위 absolute, 포인터 이벤트는 버튼에만):
  - 좌상단 칩: `이름 · Lv.5 어린` + 배고픔/행복 상태를 작은 아이콘 게이지로 (탭하면 스탯 탭 이동)
  - 우상단 칩: 지갑 (🪙 코인 / 🥬 상추)
  - 우하단 FAB 2개: 먹이주기 / 산책
- 서식지 터치 드롭, 상태 머신, `+EXP` 플로팅 등 2차 동작 그대로
- 배회 목표 좌표 산정도 그대로 (경계 클램프가 새 크기에 자동 적용) — 다만 넓어진 화면에 맞춰 `MOTION.WANDER_SPEED`/`SEEK_SPEED`는 QA에서 재튜닝 가능 (수치는 `MOTION`에만)

## 5. 실사풍 비주얼

### 5.1 달팽이 캐릭터 (인라인 SVG)

- index.html의 `.snail-motion` 안에 이모지 대신 **SVG 마크업을 직접 인라인**
- 구성: 껍질(나선 + radial gradient 셰이딩), 몸통(반투명 그라데이션), 더듬이 2쌍, 눈점, 몸 아래 미세 하이라이트
- 성장 단계 표현은 기존 `stage-*` 클래스 재사용:
  - `stage-baby`: 작게 (기존 크기 체계 유지)
  - `stage-junior`: 크게 + 껍질 나선 진하게
  - `stage-adult`: 최대 + 껍질 색상 변화 — SVG 내부 색을 CSS 변수(`--shell-color` 등)로 빼서 클래스로 제어
- 좌우 반전(`--flip`), 들썩임/기어가기/먹기 애니메이션 등 3계층 구조(entity/motion/sprite)는 그대로. `#snail-sprite`가 span→svg 래퍼로 바뀌는 것뿐
- 이동 시 실사 느낌 보강: 기어가기 애니메이션을 몸통 신축(scaleX 미세 맥동)으로 교체, 지나간 자리에 옅은 점액 자국 1~2개가 서서히 사라지는 연출 (선택 사항)

### 5.2 배경 2종 리디자인 (CSS만)

- 원색 그라데이션 → **채도 낮춘 자연 톤 다층 그라데이션 + 텍스처**
  - 텍스처: 인라인 SVG `feTurbulence` 노이즈를 `background-image`(data URI)로 겹쳐 흙/이끼 질감
  - 바닥 원근감: 아래쪽이 진한 수직 그라데이션 + 부드러운 비네트(radial-gradient)
  - 이모지 풀잎 장식(`::after`) 제거 → CSS로 그린 흐릿한 풀/돌 실루엣 레이어로 대체
- 배경 테마: `default` = **이끼 숲 바닥** (짙은 초록·흙갈색), `garden` = **햇살 정원** (따뜻한 베이지·연갈색)
- UI 톤 조정: 버튼/칩도 원색 → 차분한 톤(반투명 유리 느낌 `backdrop-filter: blur`, 미지원 브라우저는 반투명 단색 폴백)
- 상추도 이모지 대신 간단한 SVG 잎으로 교체 (낙하/축소 연출 유지)

### 5.3 파일 구조 변화

```
snail/
├── index.html          ← 5탭 구조 + 인라인 SVG (달팽이/상추)
├── css/
│   ├── theme.css       ← 자연 톤 팔레트/텍스처 변수로 개편
│   ├── components.css  ← 탭 바/칩/FAB 추가, 헤더 제거
│   ├── home.css        ← 전체 화면 서식지 + 오버레이
│   ├── stats.css       ← 신규 (스탯 화면)
│   ├── shop.css        ← 유지 (배경 선택 제거)
│   └── settings.css    ← 신규 (꾸미기/설정 공용)
└── js/
    ├── stats.js        ← 신규: StatsModule (스탯 바 렌더)
    ├── deco.js         ← 신규: DecoModule (배경 선택)
    ├── settings.js     ← 신규: SettingsModule (초기화/안내)
    └── (db/game/toast/home/shop/habitat/app 유지)
```

- 규칙 유지: 1 JS 파일 = 1 IIFE = 1 전역 네임스페이스. 스탯 바 렌더 로직은 HomeModule에서 StatsModule로 **이동**(복사 → 전환 → 삭제 순서)

## 7. 구현 단계 (브랜치 단위)

| 단계 | 브랜치 | 내용 | 완료 기준 |
|------|--------|------|-----------|
| 1 | `feat/tab-layout` | 하단 5탭 바 + 5화면 골격 + `App.navigate` 확장 (기존 콘텐츠는 임시 배치) | 탭 전환 동작, 홈 이탈 시 루프 정지 유지 |

| 3 | `feat/fullscreen-habitat` | 서식지 전체 화면 확장 + 홈 오버레이(칩/FAB) 재배치 | 넓어진 영역에서 배회/드롭/클램프 정상 |
| 4 | `feat/visual-realism` | SVG 달팽이 + 실사풍 배경/텍스처 + UI 톤 조정 | 성장 단계/반전/연출 전부 SVG에서 동작 |
| 5 | `chore/v3-qa` | 통합 테스트 갱신, 모바일 390px/safe-area QA | §8 완료 조건 전체 통과 |

## 8. 완료 조건

1. 하단 탭 바로 5개 화면이 전환되고, 중앙 홈 버튼이 시각적으로 강조된다
2. 홈에서 서식지가 상단 영역 전체를 차지하고, 달팽이가 화면 전반을 넓게 돌아다닌다
3. 스탯/상점/꾸미기/설정 각 탭에서 기존 기능(스탯 확인, 상추 구매, 배경 교체)이 그대로 동작한다
4. 달팽이가 SVG로 렌더링되고 성장 단계(크기/색), 좌우 반전, 먹기 연출이 모두 동작한다
5. 배경이 실사풍으로 바뀌고 이모지 장식이 남아 있지 않다 (지갑 등 UI 아이콘 제외)
6. 데이터 초기화는 확인 모달을 거쳐야만 실행된다
7. 기존 저장 데이터(v2)로 그대로 이어진다 — 스키마 변경 없음
8. 홈 이탈/탭 백그라운드 시 게임 루프가 멈추고 복귀 시 재개된다
9. 콘솔 에러 0개, 모바일 390px + 하단 safe-area(노치 기기)에서 탭 바가 깨지지 않는다
10. 통합 테스트(jsdom)가 새 레이아웃 기준으로 전부 통과한다

## 9. 열어둔 결정 (확인 필요)

- **5탭 구성**: `스탯 · 상점 · 홈 · 꾸미기 · 설정`으로 제안. 다른 조합(예: 산책을 별도 탭으로, 설정 대신 기록/일지)을 원하면 §3.1만 바꾸면 된다
- **달팽이 디자인 방향**: SVG 셰이딩 수준을 "일러스트에 가까운 실사풍"으로 잡았다. 사진 수준의 극실사는 SVG/CSS 한계로 3차 범위에서 제외
- 모션 수치 재튜닝 값(넓어진 서식지 대응)은 QA 단계에서 실기기 확인 후 확정
